## 1. 问题陈述

### 1.1 现状

当前 `@orbit/agent-core` 的 `ToolRegistry`（`tool-registry.ts`，~165 行）是一个纯粹的注册表容器：

```typescript
// 当前状态：空壳 ToolRegistry
export class ToolRegistry {
  private tools = new Map<string, ToolDefinition>();

  register(tool: ToolDefinition): void { /* Map.set */ }
  dispatch(name: string, args: Record<string, unknown>): Promise<string> { /* Map.get + execute */ }
}
```

这意味着：

1. **框架无法独立运行**：不注册任何工具的 agent-core 是一个"哑巴"——它能思考但不能行动。
2. **每个宿主都要重复造轮子**：Orbit Desktop、Orbit iOS、第三方集成，各自实现 `read_file`、`shell_execute`、`web_search` 等通用工具。
3. **无标准化工具接口**：不同宿主注册的同名工具行为不一致，参数签名不同，错误格式各异。
4. **无并行执行能力**：当前 dispatch 逐个串行调用，无法利用 I/O 密集型工具的并发优势。
5. **无环境自适应**：工具不检查运行环境，在缺少依赖（如 Playwright、ripgrep）时硬失败。

### 1.2 对比 hermes-agent

hermes-agent 的工具体系提供了有价值的参考：

| 维度 | hermes-agent | Orbit agent-core (当前) |
|------|-------------|----------------------|
| 内置工具数 | 50+ | 0 |
| 分组机制 | Toolset（web, file, terminal, browser, ai, memory, task, communication） | 无 |
| 可用性检查 | ✅ 运行时检测 API key / 二进制依赖 | ❌ 硬失败 |
| 并行执行 | ✅ 标记并行安全性 | ❌ 串行 |
| 渐进式加载 | ✅ 按需导入 | ❌ 不适用 |
| MCP 集成 | ✅ stdio/SSE/HTTP | ❌ 无 |
| 插件系统 | ✅ npm 包形式 | ❌ 无 |

### 1.3 目标

为 `agent-core` 提供一套**与业务无关的内置工具库**，使 agent 在纯对话模式下即可执行强大的通用任务（文件操作、搜索、终端、浏览器等），无需依赖 Orbit 特定功能。

核心设计目标：

- **零配置可用**：`import { createAgent } from '@orbit/agent-core'` 即获得 Tier 1 全部工具
- **环境自适应**：缺少 Playwright 则浏览器工具不注册，缺少 API key 则搜索工具降级
- **并行优先**：I/O 密集型工具自动并行，路径冲突自动串行
- **安全内置**：与 M3 Safety Gate 深度集成，shell/file/web 工具遵循安全约束

## 2. 设计理念

### 2.1 四大原则

1. **Toolset 分组**：工具按功能领域分组（terminal, file, web, browser, code, ai, interaction），可独立启用/禁用。每个 Toolset 是一个独立的可导入模块。

2. **可用性检查 (checkFn)**：每个工具声明运行时依赖，框架在注册前检查环境是否满足。不满足的工具不注册，不污染 LLM 的工具列表。

3. **并行执行**：工具标记是否可并行（`parallelSafe`）、是否路径作用域（`pathScoped`）。并行执行引擎自动检测冲突、分批执行。

4. **渐进式加载**：Tier 2/3 工具通过动态 `import()` 按需加载，不使用的 toolset 不增加启动开销，不引入不必要的依赖。

### 2.2 架构分层

```
┌───────────────────────────────────────────────────────┐
│                   应用层 (Orbit Desktop / CLI / SDK)    │
│   可以注册业务特定工具（对象操作、日历读写等）              │
├───────────────────────────────────────────────────────┤
│                   Tier 3: 扩展机制                      │
│   MCP Bridge · Plugin Loader · 动态工具注册              │
├───────────────────────────────────────────────────────┤
│                   Tier 2: 增强工具 (可选)                │

## 3. Toolset 分组架构

### 3.1 Toolset 概念

一个 Toolset 是一组逻辑相关的工具的集合。它定义了：

- **名称与描述**：供配置和日志使用
- **包含的工具列表**：每个工具是一个 `BuiltinTool`
- **环境要求检查**：整个 toolset 级别的可用性前置条件
- **启用/禁用控制**：用户可按 toolset 粒度控制

```typescript
// packages/agent-core/src/tools/types.ts

import type { Capability } from '../capability-types.js';

/**
 * Toolset 环境要求检查结果
 */
export interface ToolsetRequirement {
  /** 要求的名称，如 'ripgrep binary' 或 'SERPER_API_KEY' */
  readonly name: string;
  /** 是否满足 */
  readonly satisfied: boolean;
  /** 不满足时的提示信息 */
  readonly hint?: string;
  /** 是否为必需（false = 可降级运行） */
  readonly required: boolean;
}

/**
 * Toolset — 一组逻辑相关的内置工具
 *
 * 每个 Toolset 是一个独立模块，可按需导入。
 * 框架在启动时枚举所有已注册的 Toolset，检查环境要求，
 * 只注册可用的工具到 CapabilityRegistry。
 */
export interface Toolset {
  /** Toolset 唯一标识符，如 'terminal', 'file', 'web' */
  readonly name: string;

  /** 人类可读描述 */
  readonly description: string;

  /** 版本号，遵循 semver */
  readonly version: string;

  /**
   * Tier 等级：
   * - 1: 核心工具，随框架发布
   * - 2: 增强工具，可选安装
   * - 3: 扩展机制（MCP、插件）
   */
  readonly tier: 1 | 2 | 3;

  /** 此 Toolset 包含的所有工具 */
  readonly tools: readonly BuiltinTool[];

  /**
   * 检查运行环境是否满足此 Toolset 的要求。
   * 返回一个要求列表，每项标明是否满足。
   *
   * 示例：browser-tools 需要 Playwright，web-tools 需要 API key
   */
  checkRequirements(): Promise<readonly ToolsetRequirement[]>;

  /**
   * 综合判定此 Toolset 是否可用。
   * 默认逻辑：所有 required=true 的要求都 satisfied 时返回 true。
   * Toolset 可覆盖此方法提供自定义逻辑（如降级模式）。
   */
  isAvailable(): Promise<boolean>;
}
```

### 3.2 BuiltinTool 定义

`BuiltinTool` 扩展了 M2 定义的 `Capability` 接口（即 `ToolDefinition` 的升级版），增加了内置工具特有的元数据：

```typescript
// packages/agent-core/src/tools/types.ts (续)

import type { RiskLevel, ApprovalPolicy } from '../types.js';
import type { ConcurrencySafety } from '../capability-types.js';

/**
 * 工具可用性检查函数。
 * 在注册时调用，检查该工具是否在当前环境可用。
 * 返回 true 表示可用，false 表示不可用（不注册到 LLM 工具列表）。
 */
export type ToolCheckFn = () => Promise<boolean>;

/**
 * BuiltinTool — 内置工具定义，扩展 Capability 接口
 *
 * 在 M2 Capability 的基础上增加：
 * - 所属 Toolset 标识
 * - 运行时可用性检查
 * - 并行安全标记
 * - 路径作用域（用于写冲突检测）
 * - 展示用 emoji
 * - 结果大小限制
 */
export interface BuiltinTool<
  TInput extends Record<string, unknown> = Record<string, unknown>,
  TOutput = string,
> {
  // ---- 基础信息 ----
  /** 工具名称，全局唯一，如 'shell_execute', 'read_file' */
  readonly name: string;

  /** 人类可读描述（会传给 LLM） */
  readonly description: string;

  /** 所属 Toolset 名称 */
  readonly toolset: string;

  /** 展示用 emoji，用于日志和 UI */
  readonly emoji: string;

  // ---- 参数 Schema ----
  /** 输入参数的 JSON Schema（传给 LLM 的 function calling） */
  readonly inputSchema: Record<string, unknown>;

  /** 输出格式描述 */
  readonly outputFormat?: 'text' | 'json' | 'markdown' | 'binary';

  // ---- 安全声明 ----
  /** 静态风险等级（可在子类中通过输入敏感方法覆盖） */
  readonly riskLevel: RiskLevel;

  /** 静态审批策略 */
  readonly approvalPolicy: ApprovalPolicy;

  // ---- 并行控制 ----
  /**
   * 是否可安全并行执行。
   * - true: 可与其他 parallelSafe 工具并行
   * - false: 必须独占执行
   */
  readonly parallelSafe: boolean;

  /**
   * 是否为路径作用域工具。
   * 路径作用域工具在操作不同路径时可并行，操作相同路径时串行。
   * 典型：write_file（写不同文件可并行，写同一文件必须串行）
   */
  readonly pathScoped: boolean;

  /**
   * 结果最大字节数。超出时自动截断并添加 "[truncated]" 标记。
   * 默认 100KB。防止单个工具返回过大结果占满上下文窗口。
   */
  readonly maxResultSize: number;

  // ---- 运行时检查 ----
  /**
   * 运行时可用性检查。
   * 在工具注册时调用。返回 false 的工具不会注册到 CapabilityRegistry。
   *
   * 典型检查：
   * - shell_execute: 检查目标 shell 是否存在
   * - web_search: 检查 SERPER_API_KEY 或 TAVILY_API_KEY
   * - browser_navigate: 检查 Playwright 是否安装
   */
  checkFn?: ToolCheckFn;

  // ---- 执行 ----
  /**
   * 工具执行函数。
   * 接收经过 Schema 校验的输入和执行上下文。
   */
  execute(
    input: TInput,
    context: ToolExecutionContext,
  ): Promise<ToolExecutionResult<TOutput>>;
}
```

## 4. 核心接口设计

### 4.1 ToolsetRegistry

`ToolsetRegistry` 是所有内置工具的管理中心。它负责 Toolset 的注册、可用性检查、以及向 `CapabilityRegistry`（M2）的桥接注册。

```typescript
// packages/agent-core/src/tools/index.ts

import type { Toolset, BuiltinTool, ToolsetRequirement } from './types.js';
import type { CapabilityRegistry } from '../capability-registry.js';

/**
 * Toolset 注册配置
 */
export interface ToolsetRegistryConfig {
  /** 要启用的 Toolset 名称列表。为空则启用所有可用的 Tier 1 工具 */
  readonly enabledToolsets?: readonly string[];

  /** 要禁用的 Toolset 名称列表（优先级高于 enabledToolsets） */
  readonly disabledToolsets?: readonly string[];

  /** 要禁用的单个工具名称列表 */
  readonly disabledTools?: readonly string[];

  /** 是否自动注册 Tier 1 工具（默认 true） */
  readonly autoRegisterTier1?: boolean;

  /** 是否在可用性检查失败时打印警告（默认 true） */
  readonly warnOnUnavailable?: boolean;

  /** 自定义环境变量（用于覆盖 process.env） */
  readonly env?: Record<string, string>;
}

/**
 * Toolset 注册结果
 */
export interface ToolsetRegistrationResult {
  /** 已注册的 Toolset 名称列表 */
  readonly registered: readonly string[];

  /** 因不可用而跳过的 Toolset */
  readonly skipped: readonly {
    readonly name: string;
    readonly reason: string;
    readonly requirements: readonly ToolsetRequirement[];
  }[];

  /** 已注册的工具总数 */
  readonly totalTools: number;
}

export class ToolsetRegistry {
  private toolsets = new Map<string, Toolset>();
  private registeredTools = new Map<string, BuiltinTool>();

  constructor(private readonly config: ToolsetRegistryConfig = {}) {}

  /**
   * 注册一个 Toolset。
   * 不会立即检查可用性，等到 initialize() 时统一检查。
   */
  addToolset(toolset: Toolset): void {
    if (this.toolsets.has(toolset.name)) {
      throw new Error(`Toolset '${toolset.name}' is already registered`);
    }
    this.toolsets.set(toolset.name, toolset);
  }

  /**
   * 初始化：检查所有 Toolset 的可用性，将可用工具注册到 CapabilityRegistry。
   *
   * 流程：
   * 1. 枚举所有已添加的 Toolset
   * 2. 按配置过滤启用/禁用
   * 3. 并行执行 checkRequirements()
   * 4. 对可用的 Toolset，将其 tools 注册到 CapabilityRegistry
   * 5. 返回注册结果摘要
   */
  async initialize(
    capabilityRegistry: CapabilityRegistry,
  ): Promise<ToolsetRegistrationResult> {
    const registered: string[] = [];
    const skipped: ToolsetRegistrationResult['skipped'][number][] = [];

    const candidates = this.getEnabledCandidates();

    const checks = await Promise.all(
      candidates.map(async (ts) => ({
        toolset: ts,
        available: await ts.isAvailable(),
        requirements: await ts.checkRequirements(),
      })),
    );

    for (const { toolset, available, requirements } of checks) {
      if (!available) {
        const unmet = requirements.filter((r) => r.required && !r.satisfied);
        skipped.push({
          name: toolset.name,
          reason: unmet.map((r) => r.hint ?? r.name).join('; '),
          requirements,
        });
        if (this.config.warnOnUnavailable !== false) {
          console.warn(
            `[ToolsetRegistry] Skipping '${toolset.name}': ` +
              `${unmet.map((r) => r.name).join(', ')} not satisfied`,
          );
        }
        continue;
      }

      for (const tool of toolset.tools) {
        if (this.config.disabledTools?.includes(tool.name)) continue;
        if (tool.checkFn && !(await tool.checkFn())) continue;

        this.bridgeRegister(capabilityRegistry, tool);
        this.registeredTools.set(tool.name, tool);
      }

      registered.push(toolset.name);
    }

    return { registered, skipped, totalTools: this.registeredTools.size };
  }

  /** 获取所有已注册的工具 */
  getRegisteredTools(): ReadonlyMap<string, BuiltinTool> {
    return this.registeredTools;
  }

  /** 按 toolset 名称获取工具列表 */
  getToolsByToolset(toolsetName: string): readonly BuiltinTool[] {
    const ts = this.toolsets.get(toolsetName);
    if (!ts) return [];
    return ts.tools.filter((t) => this.registeredTools.has(t.name));
  }

  private getEnabledCandidates(): Toolset[] {
    const { enabledToolsets, disabledToolsets, autoRegisterTier1 } = this.config;

    return Array.from(this.toolsets.values()).filter((ts) => {
      if (disabledToolsets?.includes(ts.name)) return false;
      if (enabledToolsets && enabledToolsets.length > 0) {
        return enabledToolsets.includes(ts.name);
      }
      if (autoRegisterTier1 !== false && ts.tier === 1) return true;
      return false;
    });
  }

  private bridgeRegister(registry: CapabilityRegistry, tool: BuiltinTool): void {
    registry.register({
      name: tool.name,
      description: tool.description,
      inputSchema: tool.inputSchema,
      riskLevel: tool.riskLevel,
      approvalPolicy: tool.approvalPolicy,
      execute: async (args, ctx) => {
        const toolCtx = this.buildToolContext(ctx);
        const result = await tool.execute(args as any, toolCtx);
        return result.success ? (result.data as string) ?? '' : `Error: ${result.error}`;
      },
    });
  }

  private buildToolContext(executionCtx: any): ToolExecutionContext {
    return {
      workingDir: executionCtx.workingDir ?? process.cwd(),
      env: executionCtx.env ?? {},
      abortSignal: executionCtx.abortSignal ?? new AbortController().signal,
      onProgress: executionCtx.onProgress ?? (() => {}),
      sessionId: executionCtx.sessionId ?? '',
      runId: executionCtx.runId ?? '',
      securityConstraints: executionCtx.securityConstraints ?? DEFAULT_SECURITY,
    };
  }
}
```

### 4.2 默认初始化 — 零配置启动

```typescript
// packages/agent-core/src/tools/defaults.ts

import type { ToolsetRegistryConfig } from './index.js';
import { ToolsetRegistry } from './index.js';

// Tier 1 静态导入（随框架发布）
import { terminalToolset } from './terminal-tools.js';
import { fileToolset } from './file-tools.js';
import { webToolset } from './web-tools.js';
import { interactionToolset } from './interaction-tools.js';

/**
 * 创建预配置的 ToolsetRegistry，包含所有 Tier 1 内置工具。
 *
 * 使用方式：
 * ```typescript
 * import { createDefaultToolsetRegistry } from '@orbit/agent-core';
 *
 * const registry = createDefaultToolsetRegistry();
 * await registry.initialize(capabilityRegistry);
 * // 现在 capabilityRegistry 中已有所有可用的内置工具
 * ```
 */
export function createDefaultToolsetRegistry(
  config?: Partial<ToolsetRegistryConfig>,
): ToolsetRegistry {
  const registry = new ToolsetRegistry({
    autoRegisterTier1: true,
    warnOnUnavailable: true,
    ...config,
  });

  registry.addToolset(terminalToolset);
  registry.addToolset(fileToolset);
  registry.addToolset(webToolset);
  registry.addToolset(interactionToolset);

  return registry;
}

/**
 * 动态加载并注册 Tier 2 工具集。
 * 使用动态 import() 避免将可选依赖打入主 bundle。
 */
export async function loadOptionalToolsets(
  registry: ToolsetRegistry,
  toolsets: ('browser' | 'code' | 'ai')[],
): Promise<void> {
  for (const name of toolsets) {
    try {
      switch (name) {
        case 'browser': {
          const { browserToolset } = await import('./browser-tools.js');
          registry.addToolset(browserToolset);
          break;
        }
        case 'code': {
          const { codeToolset } = await import('./code-tools.js');

#### 5.1.1 shell_execute

执行 shell 命令，支持超时、工作目录、环境变量、stdin 输入。

```typescript
// packages/agent-core/src/tools/terminal-tools.ts

import { spawn } from 'node:child_process';
import type { BuiltinTool, ToolExecutionContext, ToolExecutionResult } from './types.js';

interface ShellExecuteInput {
  /** 要执行的命令 */
  command: string;
  /** 工作目录（相对于 context.workingDir 解析） */
  workingDir?: string;
  /** 超时时间（毫秒），默认 30000 */
  timeoutMs?: number;
  /** 额外环境变量 */
  env?: Record<string, string>;
  /** 标准输入内容 */
  stdin?: string;
}

interface ShellExecuteOutput {
  stdout: string;
  stderr: string;
  exitCode: number;
  signal?: string;
  timedOut: boolean;
}

export const shellExecuteTool: BuiltinTool<ShellExecuteInput, ShellExecuteOutput> = {
  name: 'shell_execute',
  description: [
    '执行 shell 命令并返回输出。支持超时控制、工作目录设置、环境变量注入。',
    '注意：命令在受限环境中执行，部分危险命令可能被安全策略阻止。',
    '长运行命令请使用 process_manage 替代。',
  ].join('\n'),
  toolset: 'terminal',
  emoji: '🖥️',
  inputSchema: {
    type: 'object',
    properties: {
      command: { type: 'string', description: '要执行的 shell 命令' },
      workingDir: { type: 'string', description: '工作目录（相对路径会基于当前工作区解析）' },
      timeoutMs: { type: 'number', description: '超时时间（毫秒），默认 30000', default: 30000 },
      env: { type: 'object', description: '额外环境变量', additionalProperties: { type: 'string' } },
      stdin: { type: 'string', description: '标准输入内容' },
    },
    required: ['command'],
  },
  riskLevel: 'R3-destructive',
  approvalPolicy: 'A2-confirm',
  parallelSafe: false,
  pathScoped: false,
  maxResultSize: 102400, // 100KB

  async execute(
    input: ShellExecuteInput,
    context: ToolExecutionContext,
  ): Promise<ToolExecutionResult<ShellExecuteOutput>> {
    const startTime = Date.now();
    const { securityConstraints } = context;

    // 安全检查：命令黑/白名单
    if (securityConstraints.shellDenyList.length > 0) {
      const cmd = input.command.trim().split(/\s+/)[0];
      if (securityConstraints.shellDenyList.includes(cmd)) {
        return {
          success: false,
          error: `Command '${cmd}' is blocked by security policy`,
          errorCode: 'COMMAND_DENIED',
          durationMs: Date.now() - startTime,
        };
      }
    }

    if (securityConstraints.shellAllowList.length > 0) {
      const cmd = input.command.trim().split(/\s+/)[0];
      if (!securityConstraints.shellAllowList.includes(cmd)) {
        return {
          success: false,
          error: `Command '${cmd}' is not in the allow list`,
          errorCode: 'COMMAND_NOT_ALLOWED',
          durationMs: Date.now() - startTime,
        };
      }
    }

    const timeout = input.timeoutMs ?? 30000;
    const cwd = input.workingDir
      ? resolveWorkingDir(context.workingDir, input.workingDir)
      : context.workingDir;

    return new Promise((resolve) => {
      const proc = spawn('sh', ['-c', input.command], {
        cwd,
        env: { ...context.env, ...input.env },
        timeout,
        signal: context.abortSignal,
      });

      let stdout = '';
      let stderr = '';
      let timedOut = false;

      proc.stdout?.on('data', (chunk: Buffer) => { stdout += chunk.toString(); });
      proc.stderr?.on('data', (chunk: Buffer) => { stderr += chunk.toString(); });

      if (input.stdin) {
        proc.stdin?.write(input.stdin);
        proc.stdin?.end();
      }

      proc.on('close', (exitCode, signal) => {
        resolve({
          success: exitCode === 0,
          data: {
            stdout: truncate(stdout, shellExecuteTool.maxResultSize),
            stderr: truncate(stderr, shellExecuteTool.maxResultSize / 4),
            exitCode: exitCode ?? -1,
            signal: signal ?? undefined,
            timedOut,
          },
          durationMs: Date.now() - startTime,
        });
      });

      proc.on('error', (err: NodeJS.ErrnoException) => {
        if (err.name === 'AbortError') timedOut = true;
        resolve({
          success: false,
          error: err.message,
          errorCode: timedOut ? 'TIMEOUT' : 'SPAWN_ERROR',
          durationMs: Date.now() - startTime,
        });
      });
    });
  },
};

function resolveWorkingDir(base: string, relative: string): string {
  const path = require('node:path');
  return path.isAbsolute(relative) ? relative : path.resolve(base, relative);
}

function truncate(str: string, maxBytes: number): string {
  if (Buffer.byteLength(str) <= maxBytes) return str;
  const buf = Buffer.from(str);
  return buf.subarray(0, maxBytes).toString('utf-8') + '\n[truncated]';
}

  /** 字符编码，默认 'utf-8' */
  encoding?: BufferEncoding;
}

export const readFileTool: BuiltinTool<ReadFileInput, string> = {
  name: 'read_file',
  description: [
    '读取指定文件的内容。支持按行范围读取（避免大文件一次性加载）。',
    '返回带行号前缀的文件内容。如果文件过大，建议使用 startLine/endLine 分段读取。',
  ].join('\n'),
  toolset: 'file',
  emoji: '📖',
  inputSchema: {
    type: 'object',
    properties: {
      path: { type: 'string', description: '文件路径（相对于工作目录）' },
      startLine: { type: 'number', description: '起始行号（1-based），默认 1' },
      endLine: { type: 'number', description: '结束行号（1-based），默认到文件末尾' },
      encoding: { type: 'string', description: '字符编码，默认 utf-8' },
    },
    required: ['path'],
  },
  riskLevel: 'R0-read',
  approvalPolicy: 'A0-auto',
  parallelSafe: true,
  pathScoped: true,
  maxResultSize: 102400,

  async execute(input, context) {
    const startTime = Date.now();
    const fs = await import('node:fs/promises');
    const nodePath = await import('node:path');

    const fullPath = nodePath.resolve(context.workingDir, input.path);

    if (!isPathAllowed(fullPath, context.securityConstraints)) {
      return {
        success: false,
        error: `Access denied: path '${input.path}' is outside allowed scope`,
        errorCode: 'PATH_DENIED',
        durationMs: Date.now() - startTime,
      };
    }

    try {
      const content = await fs.readFile(fullPath, input.encoding ?? 'utf-8');
      const lines = (content as string).split('\n');

      const start = (input.startLine ?? 1) - 1;
      const end = input.endLine ?? lines.length;
      const slice = lines.slice(start, end);

      const numbered = slice
        .map((line: string, i: number) => `${start + i + 1}. ${line}`)
        .join('\n');

      return {
        success: true,
        data: numbered,
        durationMs: Date.now() - startTime,
        metadata: { totalLines: lines.length, returnedLines: slice.length },
        affectedPaths: [fullPath],
      };
    } catch (err: any) {
      return {
        success: false,
        error: err.message,
        errorCode: err.code ?? 'READ_ERROR',
        durationMs: Date.now() - startTime,
      };
    }
  },
};

## 8. 并行执行引擎

### 8.1 问题

当 LLM 在单轮中调用多个工具时（如同时 `read_file` 三个文件），当前串行执行浪费了 I/O 等待时间。但盲目并行可能导致冲突（两个工具写同一个文件）。

### 8.2 ParallelExecutor 设计

```typescript
// packages/agent-core/src/tools/parallel-executor.ts

export interface ToolCallRequest {
  readonly callId: string;
  readonly toolName: string;
  readonly args: Record<string, unknown>;
}

export interface ToolCallResponse {
  readonly callId: string;
  readonly toolName: string;
  readonly result: ToolExecutionResult;
}

export interface ExecutionBatch {
  readonly index: number;
  readonly calls: readonly ToolCallRequest[];
  readonly dependsOnBatch?: number;
}

export interface ParallelExecutionPlan {
  readonly batches: readonly ExecutionBatch[];
  readonly conflicts: readonly PathConflict[];
  readonly totalBatches: number;
  readonly totalCalls: number;
}

export interface PathConflict {
  readonly path: string;
  readonly callIds: readonly string[];
  readonly reason: string;
}

/**
 * ParallelExecutor — 智能并行工具执行引擎
 *
 * 职责：
 * 1. 分析一组工具调用请求的并行性
 * 2. 检测路径冲突（两个写操作指向同一文件）
 * 3. 生成最优执行计划（最大化并行度，保证正确性）
 * 4. 按批次执行，每批次内并行、批次间串行
 * 5. 超时控制和取消传播
 */
export class ParallelExecutor {
  constructor(
    private readonly toolRegistry: ToolsetRegistry,
    private readonly config: ParallelExecutorConfig = {},
  ) {}

  /** 分析工具调用列表，生成执行计划 */
  plan(calls: readonly ToolCallRequest[]): ParallelExecutionPlan {
    if (calls.length <= 1) {
      return { batches: [{ index: 0, calls }], conflicts: [], totalBatches: 1, totalCalls: calls.length };
    }

    // 1. 分类：parallel-safe vs sequential-only
    const parallelCalls: ToolCallRequest[] = [];
    const sequentialCalls: ToolCallRequest[] = [];
    for (const call of calls) {
      const tool = this.toolRegistry.getRegisteredTools().get(call.toolName);
      (tool?.parallelSafe ? parallelCalls : sequentialCalls).push(call);
    }

    // 2. 路径冲突检测
    const { groups, conflicts } = this.detectPathConflicts(parallelCalls);

    // 3. 构建批次
    const batches: ExecutionBatch[] = [];
    if (groups.noConflict.length > 0) {
      batches.push({ index: 0, calls: groups.noConflict });
    }
    for (const group of groups.conflictGroups) {
      for (const call of group) {
        batches.push({ index: batches.length, calls: [call], dependsOnBatch: batches.length - 1 });
      }
    }
    for (const call of sequentialCalls) {
      batches.push({ index: batches.length, calls: [call], dependsOnBatch: batches.length > 0 ? batches.length - 1 : undefined });
    }

    return { batches, conflicts, totalBatches: batches.length, totalCalls: calls.length };
  }

  /** 按计划执行工具调用 */
  async execute(
    plan: ParallelExecutionPlan,
    context: ToolExecutionContext,
  ): Promise<readonly ToolCallResponse[]> {
    const results: ToolCallResponse[] = [];

    for (const batch of plan.batches) {
      if (context.abortSignal.aborted) {
        for (const call of batch.calls) {
          results.push({ callId: call.callId, toolName: call.toolName, result: { success: false, error: 'Cancelled', errorCode: 'CANCELLED', durationMs: 0 } });
        }
        continue;
      }

      context.onProgress(`Batch ${batch.index + 1}/${plan.totalBatches} (${batch.calls.length} tools)`);

      const batchResults = await Promise.all(
        batch.calls.map(async (call) => {
          const tool = this.toolRegistry.getRegisteredTools().get(call.toolName);
          if (!tool) return { callId: call.callId, toolName: call.toolName, result: { success: false, error: 'Not found', errorCode: 'TOOL_NOT_FOUND', durationMs: 0 } as ToolExecutionResult };
          const result = await this.executeWithTimeout(tool, call.args, context);
          return { callId: call.callId, toolName: call.toolName, result };
        }),
      );
      results.push(...batchResults);
    }

    return results;
  }

  private detectPathConflicts(calls: readonly ToolCallRequest[]) { /* ... 路径冲突检测逻辑 ... */ }
  private extractPaths(call: ToolCallRequest): string[] { /* ... 从参数中提取路径 ... */ }
  private async executeWithTimeout(tool: BuiltinTool, args: Record<string, unknown>, context: ToolExecutionContext): Promise<ToolExecutionResult> { /* ... */ }
}

export interface ParallelExecutorConfig {
  maxConcurrency?: number;
  detectPathConflicts?: boolean;
}
```

// packages/agent-core/src/tools/security.ts

import * as path from 'node:path';

export function isPathAllowed(fullPath: string, constraints: SecurityConstraints): boolean {
  const normalizedPath = path.normalize(fullPath);
  for (const denied of constraints.deniedPaths) {
    if (normalizedPath.startsWith(path.normalize(denied))) return false;
  }
  if (constraints.allowedPaths.length > 0) {
    return constraints.allowedPaths.some((allowed) => normalizedPath.startsWith(path.normalize(allowed)));
  }
  return true;
}

export function isUrlAllowed(url: string, blockedPatterns: readonly RegExp[], allowPrivate: boolean): boolean {
  for (const pattern of blockedPatterns) {
    if (pattern.test(url)) return allowPrivate;
  }
  return true;
}

export function isCommandAllowed(command: string, constraints: SecurityConstraints): { allowed: boolean; reason?: string } {
  const baseCmd = command.trim().split(/\s+/)[0];
  for (const denied of constraints.shellDenyList) {
    if (command.includes(denied)) return { allowed: false, reason: `Blocked pattern: '${denied}'` };
  }
  if (constraints.shellAllowList.length > 0 && !constraints.shellAllowList.includes(baseCmd)) {
    return { allowed: false, reason: `'${baseCmd}' not in allow list` };
  }
  return { allowed: true };
}
