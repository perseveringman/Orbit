## 2. 事件传输层 (Event Transport)

### 2.1 设计动机

当前 `DesktopBridge`（`apps/desktop/src/shared/contracts.ts`）仅暴露 `ping()` 与 `describeShell()`，**没有 agent 相关 IPC 方法**。前端组件通过 ViewModel 静态映射获取数据，无法接收实时事件流。

**核心问题**：agent-core 的 `AsyncGenerator<AgentEvent>` 管道停留在 Main Process，Renderer 无法消费。

### 2.2 IPC 通道设计

```
┌─────────────────────────────────────────────────────────┐
│                   IPC Channel Map                        │
├───────────────────┬─────────┬───────────────────────────┤
│ Channel           │ 方向    │ 用途                      │
├───────────────────┼─────────┼───────────────────────────┤
│ agent:event       │ M → R   │ 流式事件推送              │
│ agent:send        │ R → M   │ 用户发送消息              │
│ agent:approve     │ R → M   │ 批准能力请求              │
│ agent:reject      │ R → M   │ 拒绝能力请求              │
│ agent:cancel      │ R → M   │ 中止当前执行              │
│ agent:health      │ M → R   │ 健康状态更新              │
│ agent:sessions    │ R → M   │ 查询会话列表              │
│ agent:trace       │ R → M   │ 查询 Trace 数据           │
│ agent:metrics     │ R → M   │ 查询指标数据              │
│ agent:logs        │ R → M   │ 查询日志条目              │
└───────────────────┴─────────┴───────────────────────────┘

M → R = Main → Renderer (push via ipcRenderer.on)
R → M = Renderer → Main (invoke via ipcRenderer.invoke)
```

### 2.3 事件序列化协议

agent-core 产出的 `AgentEvent` 需经过序列化后通过 IPC 传输到 Renderer：

```typescript
// packages/agent-core/src/events.ts — M1 已定义的事件类型

/** 前端关心的事件子集 */
type FrontendAgentEvent =
  | TextDeltaEvent        // 流式文本片段
  | ToolStartEvent        // 工具开始执行
  | ToolProgressEvent     // 工具进度更新
  | ToolCompleteEvent     // 工具执行完成
  | ApprovalRequestEvent  // 审批请求
  | ApprovalResolvedEvent // 审批结果
  | RunStatusEvent        // 运行状态变更
  | StageChangeEvent      // 阶段切换 (routing → assembly → thinking → ...)
  | TokenUsageEvent       // Token 消耗更新
  | ErrorEvent;           // 错误事件

/** IPC 传输信封 */
interface AgentEventEnvelope {
  /** 单调递增序号，用于断线重连后补发 */
  seq: number;
  /** ISO-8601 时间戳 */
  timestamp: string;
  /** 关联的 sessionId */
  sessionId: string;
  /** 事件载荷 */
  payload: FrontendAgentEvent;
}
```

**设计决策**：

- **序号字段 `seq`**：Renderer 断线重连时携带 `lastSeq`，Main Process 从 EventBuffer 中补发缺失事件，避免状态不一致。
- **大结果截断**：工具返回结果 > 10 KB 时截断为前 2 KB + `"... (truncated, 查看完整)"` 标记，完整结果通过按需查询获取。
- **批量发送**：高频 `text-delta` 事件在 Main Process 中以 16ms 为窗口合并为一个 envelope，降低 IPC 开销。

### 2.4 Main Process 事件路由

```typescript
// apps/desktop/src/main/agent-host.ts

import { ipcMain, BrowserWindow } from 'electron';
import type { AgentEventEnvelope, FrontendAgentEvent } from '@orbit/agent-core';

/**
 * EventBuffer — 环形缓冲区，用于断线重连补发。
 * 容量 1000 条，FIFO 淘汰。
 */
class EventBuffer {
  private buffer: AgentEventEnvelope[] = [];
  private seq = 0;
  private readonly capacity: number;

  constructor(capacity = 1000) {
    this.capacity = capacity;
  }

  push(sessionId: string, payload: FrontendAgentEvent): AgentEventEnvelope {
    const envelope: AgentEventEnvelope = {
      seq: ++this.seq,
      timestamp: new Date().toISOString(),
      sessionId,
      payload,
    };
    this.buffer.push(envelope);
    if (this.buffer.length > this.capacity) {
      this.buffer.shift();
    }
    return envelope;
  }

  since(lastSeq: number): AgentEventEnvelope[] {
    return this.buffer.filter((e) => e.seq > lastSeq);
  }
}

/**
 * AgentHost — 在 Main Process 中运行，桥接 agent-core 与 Renderer。
 *
 * 职责：
 * 1. 订阅 agent-core EventBus，将事件推送到所有 BrowserWindow
 * 2. 监听 Renderer IPC 调用，转发为 agent-core 命令
 * 3. 维护 EventBuffer 支持断线补发
 */
class AgentHost {
  private readonly buffer = new EventBuffer();
  private readonly windows = new Set<BrowserWindow>();

  /** 注册一个 BrowserWindow 接收事件推送 */
  attach(win: BrowserWindow): void {
    this.windows.add(win);
    win.on('closed', () => this.windows.delete(win));
  }

  /** 启动 IPC 监听 */
  listen(): void {
    // R → M: 用户发送消息
    ipcMain.handle('agent:send', async (_event, sessionId: string, content: string) => {
      return this.handleSendMessage(sessionId, content);
    });

    // R → M: 审批操作
    ipcMain.handle('agent:approve', async (_event, approvalId: string) => {
      return this.handleApprove(approvalId);
    });

    ipcMain.handle('agent:reject', async (_event, approvalId: string, reason?: string) => {
      return this.handleReject(approvalId, reason);
    });

    // R → M: 中止执行
    ipcMain.handle('agent:cancel', async (_event, sessionId: string) => {
      return this.handleCancel(sessionId);
    });

    // R → M: 查询类接口
    ipcMain.handle('agent:sessions', async (_event, filter?: SessionFilter) => {
      return this.handleGetSessions(filter);
    });

    ipcMain.handle('agent:trace', async (_event, sessionId: string) => {
      return this.handleGetTrace(sessionId);
    });

    ipcMain.handle('agent:metrics', async (_event, query: MetricsQuery) => {
      return this.handleGetMetrics(query);
    });

    ipcMain.handle('agent:logs', async (_event, query: LogQuery) => {
      return this.handleGetLogs(query);
    });
  }

  /** 将 agent-core 事件广播到所有窗口 */
  broadcast(sessionId: string, payload: FrontendAgentEvent): void {
    const envelope = this.buffer.push(sessionId, payload);
    for (const win of this.windows) {
      if (!win.isDestroyed()) {
        win.webContents.send('agent:event', envelope);
      }
    }
  }

  // ... handler implementations delegate to agent-core
}
```

### 2.5 事件批量合并 (Text Delta Coalescing)

```typescript
// apps/desktop/src/main/agent-host.ts (续)

/**
 * TextDeltaCoalescer — 将高频 text-delta 事件合并发送。
 *
 * LLM 流式输出约每 20-50ms 产出一个 token，直接逐个通过 IPC 发送
 * 会造成 Renderer 频繁 re-render。合并为 16ms 窗口内的单个事件。
 *
 * ┌─────┐  ┌─────┐  ┌─────┐     ┌───────────────────┐
 * │ "He"│  │"llo"│  │" wo"│ ──▶ │ "Hello wo"        │  (一次 IPC)
 * └─────┘  └─────┘  └─────┘     └───────────────────┘
 *   0ms      8ms      14ms        16ms flush
 */
class TextDeltaCoalescer {
  private pending = '';
  private timer: ReturnType<typeof setTimeout> | null = null;
  private readonly flushIntervalMs: number;
  private readonly onFlush: (merged: string) => void;

  constructor(onFlush: (merged: string) => void, flushIntervalMs = 16) {
    this.onFlush = onFlush;
    this.flushIntervalMs = flushIntervalMs;
  }

  push(delta: string): void {
    this.pending += delta;
    if (!this.timer) {
      this.timer = setTimeout(() => this.flush(), this.flushIntervalMs);
    }
  }

  flush(): void {
    if (this.pending) {
      this.onFlush(this.pending);
      this.pending = '';
    }
    if (this.timer) {
      clearTimeout(this.timer);
      this.timer = null;
    }
  }
}
```

## 3. 实时执行可视化

### 3.1 流式消息渲染 — StreamingMessage

#### 3.1.1 设计动机

当前 `MessageBubble` 组件一次性渲染完整消息内容。流式场景下需要 token-by-token 的打字机效果，并在文本不完整时也能进行渐进式 Markdown 渲染。

#### 3.1.2 组件接口

```typescript
// packages/feature-workbench/src/agent/StreamingMessage.tsx

interface StreamingMessageProps {
  /** 已积累的文本（每次 text-delta 追加后的完整字符串） */
  content: string;
  /** 是否仍在流式接收中 */
  isStreaming: boolean;
  /** 角色标签 (用于左侧标识) */
  roleLabel: string;
  /** thinking tokens（折叠展示） */
  thinkingContent?: string;
  /** 消息 token 数 */
  tokenCount?: number;
  /** 累计成本 USD */
  cost?: number;
}
```

#### 3.1.3 渲染策略

```

│  助手                                            12:03   │
│ ─────────────────────────────────────────────────────── │
│                                                          │

│                                                          │

**关键实现细节**：

1. **Markdown 渐进渲染**：使用增量 Markdown parser，在未闭合的代码块场景下自动补全围栏（trailing ` ``` `），避免渲染错乱。
2. **光标指示器**：流式接收中在文本末尾显示 `▍` 闪烁光标。
3. **代码块高亮**：增量式语法高亮，每次追加文本时只重新高亮当前代码块，非整体重渲染。
4. **Thinking 折叠**：thinking tokens 默认折叠为 `💭 thinking (N tokens)` 一行，点击展开查看推理过程。
5. **Token/成本显示**：消息底部右对齐显示，灰色小字。

#### 4.2.4 超时倒计时逻辑

```typescript
// 倒计时 hook
function useCountdown(totalSeconds: number): {
  remaining: number;
  percentage: number;
  isUrgent: boolean;  // < 60s
  isExpired: boolean;
} {
  const [remaining, setRemaining] = useState(totalSeconds);

  useEffect(() => {
    const timer = setInterval(() => {
      setRemaining((prev) => Math.max(0, prev - 1));
    }, 1000);
    return () => clearInterval(timer);
  }, [totalSeconds]);

  return {
    remaining,
    percentage: (remaining / totalSeconds) * 100,
    isUrgent: remaining < 60,
    isExpired: remaining === 0,
  };
}
```

**超时行为**：倒计时归零后自动发送 `agent:reject`，显示 "已超时自动拒绝" 提示。默认超时 5 分钟，可在设置中配置。

/**
 * useAgentEvents — 核心 Hook，消费 AgentBridge 事件流，
 * 将事件 reduce 为 UI 状态。
 */
function useAgentEvents(sessionId: string): {
  state: AgentUIState;
  actions: {
    sendMessage: (content: string) => void;
    approve: (approvalId: string) => void;
    reject: (approvalId: string, reason?: string) => void;
    alwaysAllow: (approvalId: string) => void;
    cancel: () => void;
  };
} {
  const [state, dispatch] = useReducer(agentEventReducer, initialState);

  useEffect(() => {
    const bridge = window.orbitDesktop?.agent;
    if (!bridge) return;

    const unsubscribe = bridge.onEvent((envelope) => {
      dispatch({ type: 'AGENT_EVENT', payload: envelope.payload });
    });

    return unsubscribe;
  }, [sessionId]);

  const actions = useMemo(() => ({
    sendMessage: (content: string) => {
      window.orbitDesktop?.agent.sendMessage(sessionId, content);
      dispatch({ type: 'MESSAGE_SENT', payload: content });
    },
    approve: (approvalId: string) => {
      window.orbitDesktop?.agent.approve(approvalId);
      dispatch({ type: 'APPROVAL_RESOLVED', payload: { approvalId, outcome: 'approved' } });
    },
    reject: (approvalId: string, reason?: string) => {
      window.orbitDesktop?.agent.reject(approvalId, reason);
      dispatch({ type: 'APPROVAL_RESOLVED', payload: { approvalId, outcome: 'rejected', reason } });
    },
    alwaysAllow: (approvalId: string) => {
      window.orbitDesktop?.agent.approve(approvalId);
      dispatch({ type: 'APPROVAL_RESOLVED', payload: { approvalId, outcome: 'always-allow' } });
    },
    cancel: () => {
      window.orbitDesktop?.agent.cancel(sessionId);
      dispatch({ type: 'CANCELLED' });
    },
  }), [sessionId]);

  return { state, actions };
}
```

### 7.3 事件 Reducer 核心逻辑

```typescript
// packages/feature-workbench/src/agent/agent-event-reducer.ts

type AgentUIAction =
  | { type: 'AGENT_EVENT'; payload: FrontendAgentEvent }
  | { type: 'MESSAGE_SENT'; payload: string }
  | { type: 'APPROVAL_RESOLVED'; payload: { approvalId: string; outcome: string; reason?: string } }
  | { type: 'CANCELLED' };

function agentEventReducer(state: AgentUIState, action: AgentUIAction): AgentUIState {
  switch (action.type) {
    case 'AGENT_EVENT':
      return reduceAgentEvent(state, action.payload);
    case 'MESSAGE_SENT':
      return { ...state, streamingContent: '', isStreaming: false };
    case 'CANCELLED':
      return { ...state, isStreaming: false, currentStage: 'routing' };
    default:
      return state;
  }
}

function reduceAgentEvent(state: AgentUIState, event: FrontendAgentEvent): AgentUIState {
  switch (event.kind) {
    case 'text-delta':
      return {
        ...state,
        streamingContent: state.streamingContent + event.delta,
        isStreaming: true,
      };

    case 'tool-start': {
      const newTools = new Map(state.toolStates);
      newTools.set(event.toolCallId, {
        toolCallId: event.toolCallId,
        toolName: event.toolName,
        emoji: getToolEmoji(event.toolName),
        preview: buildToolPreview(event.toolName, event.args),
        status: 'running',
        args: event.args,
        startTime: Date.now(),
      });
      return { ...state, toolStates: newTools };
    }

    case 'tool-complete': {
      const newTools = new Map(state.toolStates);
      const existing = newTools.get(event.toolCallId);
      if (existing) {
        newTools.set(event.toolCallId, {
          ...existing,
          status: event.error ? 'error' : 'success',
          result: event.result,
          error: event.error,
          durationMs: Date.now() - existing.startTime,
        });
      }
      return { ...state, toolStates: newTools };
    }

    case 'approval-request':
      return {
        ...state,
        pendingApprovals: [...state.pendingApprovals, mapApprovalEvent(event)],
      };

    case 'stage-change':
      return { ...state, currentStage: event.stage };

    case 'token-usage':
      return {
        ...state,
        progress: {
          ...state.progress,
          tokensUsed: event.totalTokens,
          totalCost: event.totalCost,
          iteration: event.iteration ?? state.progress.iteration,
        },
      };

    // ... 其余事件类型
    default:
      return state;
  }
}

/**
 * AgentBridge — Renderer 层访问 agent 服务的完整 API。
 *
 * 设计原则:
 * - contextIsolation 安全: 所有方法通过 contextBridge.exposeInMainWorld 暴露
 * - 最小权限: Renderer 只能发送预定义的操作，不能直接访问 Node.js API
 * - 事件驱动: Main → Renderer 推送通过回调订阅，非轮询
 */
interface AgentBridge {
  // ─── Outbound (Renderer → Main) ───

  /** 发送用户消息 */
  sendMessage(sessionId: string, content: string): void;

  /** 批准能力请求 */
  approve(approvalId: string): void;

  /** 拒绝能力请求 */
  reject(approvalId: string, reason?: string): void;

  /** 中止当前执行 */
  cancel(sessionId: string): void;

  // ─── Inbound (Main → Renderer) ───

  /** 订阅 agent 事件流，返回取消订阅函数 */
  onEvent(callback: (event: AgentEventEnvelope) => void): () => void;

  /** 订阅健康状态更新 */
  onHealthUpdate(callback: (status: HealthCheckResult) => void): () => void;

  // ─── Query (Renderer → Main, async) ───

  /** 查询会话列表 */
  getSessions(filter?: SessionFilter): Promise<SessionSummary[]>;

  /** 查询会话 Trace 数据 */
  getSessionTrace(sessionId: string): Promise<TraceData>;

  /** 查询指标数据 */
  getMetrics(query: MetricsQuery): Promise<SessionMetrics>;

  /** 查询日志条目 */
  getLogs(query: LogQuery): Promise<LogEntry[]>;

  /** 执行健康检查 */
  runHealthCheck(): Promise<HealthCheckResult>;

  /** 获取消息历史（含 system prompt，用于 MessageInspector） */
  getMessageHistory(sessionId: string): Promise<InspectorMessage[]>;
}

### 8.3 Preload 实现

```typescript
// apps/desktop/src/preload/agent-bridge.ts

import { contextBridge, ipcRenderer } from 'electron';
import type { AgentBridge, AgentEventEnvelope, HealthCheckResult } from '../shared/contracts';

const agentBridge: AgentBridge = {
  // Outbound
  sendMessage: (sessionId, content) =>
    ipcRenderer.send('agent:send', sessionId, content),

  approve: (approvalId) =>
    ipcRenderer.send('agent:approve', approvalId),

  reject: (approvalId, reason) =>
    ipcRenderer.send('agent:reject', approvalId, reason),

  cancel: (sessionId) =>
    ipcRenderer.send('agent:cancel', sessionId),

  // Inbound
  onEvent: (callback) => {
    const handler = (_event: Electron.IpcRendererEvent, envelope: AgentEventEnvelope) => {
      callback(envelope);
    };
    ipcRenderer.on('agent:event', handler);
    return () => ipcRenderer.removeListener('agent:event', handler);
  },

  onHealthUpdate: (callback) => {
    const handler = (_event: Electron.IpcRendererEvent, status: HealthCheckResult) => {
      callback(status);
    };
    ipcRenderer.on('agent:health', handler);
    return () => ipcRenderer.removeListener('agent:health', handler);
  },

  // Query
  getSessions: (filter) => ipcRenderer.invoke('agent:sessions', filter),
  getSessionTrace: (sessionId) => ipcRenderer.invoke('agent:trace', sessionId),
  getMetrics: (query) => ipcRenderer.invoke('agent:metrics', query),
  getLogs: (query) => ipcRenderer.invoke('agent:logs', query),
  runHealthCheck: () => ipcRenderer.invoke('agent:health-check'),
  getMessageHistory: (sessionId) => ipcRenderer.invoke('agent:messages', sessionId),
};

/** 注册到 window.orbitDesktop.agent */
export function exposeAgentBridge(): void {
  // 合并到现有 orbitDesktop 对象
  contextBridge.exposeInMainWorld('orbitDesktopAgent', agentBridge);
}
```

### 8.4 安全性考量

| 考量点 | 措施 |
|--------|------|
| **contextIsolation** | 保持 `true`，所有 API 通过 `contextBridge` 暴露 |
| **参数验证** | Main Process 端校验所有 IPC 参数类型与范围 |
| **频率限制** | `agent:send` 限制 10 msg/s，防止 Renderer 洪泛 |
| **结果截断** | 大 payload > 10KB 在 Main Process 端截断后发送 |
| **通道白名单** | Preload 只暴露 `agent:*` 前缀通道 |

## 12. 与其他里程碑的关系

```
  M0.5 流式输出 ─────────┐
                          │  enables token-by-token rendering
  M1 事件流管道 ──────────┤
                          │  primary data source for all UI
  M3 SafetyGate ──────────┤
                          ├──▶ M10 前端集成层
  M5 会话谱系 ────────────┤
                          │  context snapshots for replay
  M9 可观测性 ────────────┘
                             feeds DevTools panels
```

| 依赖 | 关系 | 说明 |
|------|------|------|
| **M1 → M10** | 🔴 强依赖 | M1 的 `AgentEvent` 流是所有可视化的数据源 |
| **M0.5 → M10** | 🔴 强依赖 | 流式输出能力驱动 `StreamingMessage` 打字机效果 |
| **M3 → M10** | 🟡 中依赖 | SafetyGate 的 `ApprovalRequest` 驱动审批 UI |
| **M9 → M10** | 🟡 中依赖 | Span/Metric/Log 数据驱动 DevTools 面板 |
| **M5 → M10** | 🔵 弱依赖 | 上下文快照支持会话回放（P2 功能） |
| **M4 → M10** | 🔵 弱依赖 | 记忆系统状态可在 DevTools 中展示 |
