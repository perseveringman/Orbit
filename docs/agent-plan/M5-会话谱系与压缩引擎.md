> **里程碑状态**：Draft  
> **依赖**：M1（事件流）、M3（Safety Gate）、M4（记忆系统）  
> **包范围**：`@orbit/agent-core`（主要变更）、`@orbit/db-schema`（新表）  
> **预计文件变更**：~15 个文件新增/修改

## 0. 问题总览与设计原则

### 当前痛点

| # | 问题 | 影响 | 现状代码 |
|---|------|------|----------|
| P1 | Session 无持久化 | 浏览器刷新/App 重启后会话丢失 | `AgentSession` 仅为内存接口 |
| P2 | Lineage 仅内存 | 无法跨会话续接（"三天后回到研究空间"） | `lineage: readonly SessionLineage[]` 无存储 |
| P3 | 压缩单级 | 中间段全部 summarize，无法区分轻重缓急 | `ContextCompressor.compress()` 只做 head/tail/middle |
| P4 | 无恢复预算 | 压缩后丢失回读关键对象的能力 | 压缩后无 token 预留 |
| P5 | 无上下文告警 | token 超限时静默失败 | 无 warning/error/autoCompact 机制 |
| P6 | Token 粗估 | `chars/4` 对中文/特殊字符偏差 30%+ | `estimateTokens()` 单一实现 |

### 设计原则

1. **渐进式压缩**：从零成本操作开始，逐级引入 LLM；每级压缩都可独立回退
2. **谱系即溯源**：任何会话的任何状态变更都可通过 lineage 链追溯到根源
3. **预算守恒**：压缩释放的空间必须有明确的再分配策略（恢复预算）
4. **Local-First**：所有持久化走客户端 SQLite，与 Orbit 现有 `@orbit/db-schema` 模式一致
5. **可测试性**：每个组件都有纯内存实现，方便单元测试

## 1. 会话持久化与谱系管理

### 1.1 SQLite 表结构设计

在 `@orbit/db-schema` 的 `ORBIT_SQLITE_TABLES` 中新增三张表，遵循现有的 `SqliteTableDef` 模式：

```typescript
// packages/db-schema/src/index.ts 新增

export const ORBIT_SQLITE_TABLES = {
  // ... 现有 11 张表 ...
  agentSessions: 'agent_sessions',
  sessionLineages: 'session_lineages',
  sessionSummaries: 'session_summaries',
} as const;
```

#### 表 1：`agent_sessions`

存储会话元数据和完整消息历史。消息以 JSON 数组存储，避免额外的 messages 表带来的 join 开销（单会话消息量有限，通常 < 200 条）。

```sql
CREATE TABLE IF NOT EXISTS agent_sessions (
  id                TEXT PRIMARY KEY,            -- generateId('ses') 格式
  workspace_id      TEXT NOT NULL,               -- FK → workspaces.id
  surface           TEXT NOT NULL,               -- AgentSurface 枚举值
  anchor_object_ids TEXT NOT NULL DEFAULT '[]',  -- JSON array of object IDs
  status            TEXT NOT NULL DEFAULT 'active', -- active|paused|completed|failed
  messages_json     TEXT NOT NULL DEFAULT '[]',  -- JSON array of AgentMessage
  token_count       INTEGER NOT NULL DEFAULT 0,  -- 最后一次精确/估算的 token 总量
  compression_level INTEGER NOT NULL DEFAULT 0,  -- 0=无 1=微压缩 2=turn 3=session 4=workspace 5=handoff
  end_reason        TEXT,                        -- 'user_end'|'auto_compact'|'handoff'|'error'|NULL
  created_at        TEXT NOT NULL,               -- ISO 8601
  updated_at        TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_agent_sessions_workspace
  ON agent_sessions (workspace_id, status);
CREATE INDEX IF NOT EXISTS idx_agent_sessions_surface
  ON agent_sessions (workspace_id, surface);
CREATE INDEX IF NOT EXISTS idx_agent_sessions_updated
  ON agent_sessions (updated_at DESC);
```

**设计决策**：
- `messages_json` 用 TEXT 存储 JSON 而非独立表：Orbit 是 local-first 应用，单会话消息数有上限，JSON 列简化了读写逻辑，且与现有 `blobs` 表的设计哲学一致
- `token_count` 缓存避免每次组装上下文都重新计算
- `compression_level` 记录当前会话经历过的最高压缩级别
- `end_reason` 参考 Hermes 的 compression 结束原因标记

#### 表 2：`session_lineages`

独立的谱系关系表，支持多对多（一个 session 可以有多个 lineage 来源）。

```sql
CREATE TABLE IF NOT EXISTS session_lineages (
  id         TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,            -- 当前 session
  type       TEXT NOT NULL,            -- LineageType 枚举值
  source_id  TEXT NOT NULL,            -- 来源 session/object/approval/job ID
  metadata   TEXT DEFAULT '{}',        -- JSON: 额外上下文（如压缩时保存 token_saved）
  created_at TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_session_lineages_session
  ON session_lineages (session_id);
CREATE INDEX IF NOT EXISTS idx_session_lineages_source
  ON session_lineages (source_id);
CREATE INDEX IF NOT EXISTS idx_session_lineages_type
  ON session_lineages (session_id, type);
```

**Lineage 类型语义**：

| type | source_id 含义 | 使用场景 |
|------|----------------|----------|
| `continues_from` | 前一个 session ID | 用户手动续接 / 自动续接 |
| `delegated_from` | 父 run ID | Orchestrator.delegate() |
| `spawned_by_object` | 对象 ID (kind:id) | 从对象上下文启动 agent |
| `blocked_by_approval` | approval request ID | 安全审批暂停 |
| `resumed_from_job` | 异步 job ID | 后台任务完成后恢复 |
| `compressed_into` | 新 session ID | 压缩产生的 handoff session |

#### 表 3：`session_summaries`

独立存储每次压缩产生的摘要，支持多级摘要复用和增量更新。

```sql
CREATE TABLE IF NOT EXISTS session_summaries (
  id               TEXT PRIMARY KEY,
  session_id       TEXT NOT NULL,       -- 被摘要的 session
  level            INTEGER NOT NULL,    -- 压缩级别: 1-5
  summary_text     TEXT NOT NULL,       -- 摘要内容
  dimensions_json  TEXT,                -- 仅 level=3 时：11维分析 JSON
  token_count      INTEGER NOT NULL,    -- 摘要本身的 token 数
  source_range     TEXT,                -- 被摘要的消息范围 "from_idx:to_idx"
  parent_summary_id TEXT,              -- 增量更新时指向上一次摘要
  created_at       TEXT NOT NULL
);

CREATE INDEX IF NOT EXISTS idx_session_summaries_session
  ON session_summaries (session_id, level);
```

```typescript
// packages/agent-core/src/sqlite-session-store.ts

export class SqliteSessionStore implements SessionStore {
  constructor(private readonly db: DatabasePort) {}

  async save(session: PersistedSession): Promise<void> {
    this.db.exec(`
      INSERT INTO agent_sessions
        (id, workspace_id, surface, anchor_object_ids, status,
         messages_json, token_count, compression_level, end_reason,
         created_at, updated_at)
      VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?, ?, ?)
    `, [
      session.id, session.workspaceId, session.surface,
      JSON.stringify(session.anchorObjectIds),
      session.status,
      JSON.stringify(session.messages),
      session.tokenCount,
      session.compressionLevel,
      session.endReason ?? null,
      session.createdAt, session.updatedAt,
    ]);
  }

  async get(id: string): Promise<PersistedSession | null> {
    const row = this.db.get('SELECT * FROM agent_sessions WHERE id = ?', [id]);
    return row ? this.rowToSession(row) : null;
  }

  // 谱系查询 - 支持正向和反向查询
  async getLineage(query: LineageQuery): Promise<readonly (SessionLineage & { metadata?: Record<string, unknown> })[]> {
    const conditions: string[] = [];
    const params: unknown[] = [];

    if (query.sessionId) {
      conditions.push('session_id = ?');
      params.push(query.sessionId);
    }
    if (query.sourceId) {
      conditions.push('source_id = ?');
      params.push(query.sourceId);
    }
    if (query.type) {
      conditions.push('type = ?');
      params.push(query.type);
    }

    const where = conditions.length > 0 ? `WHERE ${conditions.join(' AND ')}` : '';
    const rows = this.db.all(`SELECT * FROM session_lineages ${where} ORDER BY created_at`, params);

    return rows.map(row => ({
      type: row.type as LineageType,
      sourceId: row.source_id as string,
      metadata: row.metadata ? JSON.parse(row.metadata as string) : undefined,
    }));
  }

  private rowToSession(row: Record<string, unknown>): PersistedSession {
    return {
      id: row.id as string,
      workspaceId: row.workspace_id as string,
      surface: row.surface as AgentSurface,
      anchorObjectIds: JSON.parse(row.anchor_object_ids as string),
      lineage: [],  // 从 session_lineages 表单独加载
      status: row.status as SessionStatus,
      messages: JSON.parse(row.messages_json as string),
      tokenCount: row.token_count as number,
      compressionLevel: row.compression_level as CompressionLevel,
      endReason: row.end_reason as string | undefined,
      createdAt: row.created_at as string,
      updatedAt: row.updated_at as string,
    };
  }
}
```

### 1.3 谱系图遍历

实现从任意会话回溯到根会话的谱系链，以及查找所有后代会话的能力。

```typescript
// packages/agent-core/src/lineage-graph.ts

export interface LineageNode {
  readonly sessionId: string;
  readonly session: PersistedSession | null;  // null = 已删除/未加载
  readonly lineageType?: LineageType;         // 与父节点的关系
  readonly children: readonly LineageNode[];
  readonly depth: number;
}

export class LineageGraph {
  constructor(private readonly store: SessionStore) {}

  /**
   * 从当前 session 回溯到根 session。
   * 采用 BFS 沿 continues_from / compressed_into 的反向链回溯。
   * 最大深度限制 50 防止无限循环。
   */
  async traceToRoot(sessionId: string): Promise<readonly PersistedSession[]> {
    const chain: PersistedSession[] = [];
    const visited = new Set<string>();
    let currentId: string | null = sessionId;
    const MAX_DEPTH = 50;

    while (currentId && !visited.has(currentId) && chain.length < MAX_DEPTH) {
      visited.add(currentId);
      const session = await this.store.get(currentId);
      if (!session) break;

      chain.unshift(session);  // 根在前，当前在后

      // 查找父谱系（continues_from 或 compressed_into 的源）
      const lineages = await this.store.getLineage({ sessionId: currentId });
      const parent = lineages.find(l =>
        l.type === 'continues_from' || l.type === 'compressed_into'
      );
      currentId = parent?.sourceId ?? null;
    }

    return chain;
  }

  /**
   * 获取某个 session 的所有直接子 session。
   */
  async getChildren(sessionId: string): Promise<readonly PersistedSession[]> {
    // 查找所有 sourceId = sessionId 的谱系记录
    const lineages = await this.store.getLineage({ sourceId: sessionId });
    const childIds = [...new Set(lineages.map(l => l.sourceId))];

    // 注意：此处需要反查，找出哪些 session 的 lineage.sourceId 指向 sessionId
    // 实际实现需要一个反向索引查询
    const children: PersistedSession[] = [];
    for (const lineage of lineages) {
      // lineage 记录在子 session 的 session_lineages 表中
      // 需要根据 source_id = sessionId 查到子 session_id
    }
    return children;
  }

  /**
   * 构建完整谱系树（从根到所有叶子）。
   * 用于 UI 展示和调试。
   */
  async buildTree(rootSessionId: string): Promise<LineageNode> {
    return this.buildNodeRecursive(rootSessionId, undefined, 0, new Set());
  }

  private async buildNodeRecursive(
    sessionId: string,
    lineageType: LineageType | undefined,
    depth: number,
    visited: Set<string>,
  ): Promise<LineageNode> {
    if (visited.has(sessionId) || depth > 20) {
      return { sessionId, session: null, lineageType, children: [], depth };
    }
    visited.add(sessionId);

    const session = await this.store.get(sessionId);
    const children = await this.getChildren(sessionId);

    const childNodes = await Promise.all(
      children.map(async (child) => {
        const childLineages = await this.store.getLineage({ sessionId: child.id });
        const rel = childLineages.find(l => l.sourceId === sessionId);
        return this.buildNodeRecursive(child.id, rel?.type, depth + 1, visited);
      }),
    );

    return { sessionId, session, lineageType, children: childNodes, depth };
  }
}
```

### 1.4 跨会话续接流程

跨会话续接是 M5 的核心用户场景：用户三天后回到研究空间，系统自动检测相关历史会话，加载摘要，注入上下文。

```typescript
// packages/agent-core/src/session-resumption.ts

export interface ResumptionCandidate {
  readonly session: PersistedSession;
  readonly relevanceScore: number;       // 0-1
  readonly matchReason: string;
  readonly summary: SessionSummaryRecord | null;
}

export interface ResumptionContext {
  readonly candidates: readonly ResumptionCandidate[];
  readonly selectedSession: PersistedSession | null;
  readonly injectedSummary: string;      // 注入上下文窗口的文本
  readonly tokenBudget: number;          // 分配给恢复内容的 token 预算
}

export class SessionResumptionService {
  constructor(
    private readonly store: SessionStore,
    private readonly lineageGraph: LineageGraph,
    private readonly compressor: CompressionEngine,  // 后面定义
  ) {}

  /**
   * 检测可续接的历史会话。
   * 评分规则：
   *   - 同 surface 同 workspace：+0.3
   *   - anchor_object_ids 重叠：+0.3 * 重叠率
   *   - 时间衰减：exp(-daysSince / 30)，30天半衰期
   *   - 有谱系关联：+0.2
   */
  async findCandidates(
    workspaceId: string,
    surface: string,
    anchorObjectIds: readonly string[],
    limit: number = 5,
  ): Promise<readonly ResumptionCandidate[]> {
    // 1. 查询同 workspace 下最近的已完成/暂停会话
    const sessions = await this.store.listByWorkspace(workspaceId, {
      status: 'completed',
      limit: 50,
    });

    const pausedSessions = await this.store.listByWorkspace(workspaceId, {
      status: 'paused',
      limit: 20,
    });

    const allCandidates = [...sessions, ...pausedSessions];

    // 2. 计算相关性评分
    const scored: ResumptionCandidate[] = [];
    const now = Date.now();

    for (const session of allCandidates) {
      let score = 0;
      const reasons: string[] = [];

      // 同 surface 加分
      if (session.surface === surface) {
        score += 0.3;
        reasons.push('same_surface');
      }

      // anchor 对象重叠
      const overlap = session.anchorObjectIds.filter(
        id => anchorObjectIds.includes(id)
      );
      if (overlap.length > 0) {
        const overlapRate = overlap.length / Math.max(anchorObjectIds.length, 1);
        score += 0.3 * overlapRate;
        reasons.push(`anchor_overlap(${overlap.length})`);
      }

      // 时间衰减
      const daysSince = (now - new Date(session.updatedAt).getTime()) / 86400000;
      const timeScore = Math.exp(-daysSince / 30) * 0.2;
      score += timeScore;

      // 有谱系关联
      const lineages = await this.store.getLineage({ sessionId: session.id });
      if (lineages.length > 0) {
        score += 0.2;
        reasons.push('has_lineage');
      }

      const summary = await this.store.getLatestSummary(session.id);

      scored.push({
        session,
        relevanceScore: Math.min(score, 1.0),
        matchReason: reasons.join(', '),
        summary,
      });
    }

    // 3. 按评分降序排列
    scored.sort((a, b) => b.relevanceScore - a.relevanceScore);
    return scored.slice(0, limit);
  }

  /**
   * 执行会话续接：
   * 1. 选择最佳候选
   * 2. 加载其最新摘要
   * 3. 构建注入上下文
   * 4. 创建新 session 并建立 continues_from 谱系
   */
  async resume(
    workspaceId: string,
    surface: AgentSurface,
    anchorObjectIds: readonly string[],
    recoveryBudgetTokens: number = 50_000,
  ): Promise<ResumptionContext> {
    const candidates = await this.findCandidates(
      workspaceId, surface, anchorObjectIds,
    );

    if (candidates.length === 0) {
      return {
        candidates: [],
        selectedSession: null,
        injectedSummary: '',
        tokenBudget: 0,
      };
    }

    const best = candidates[0];
    if (best.relevanceScore < 0.3) {
      // 相关性太低，不自动续接
      return { candidates, selectedSession: null, injectedSummary: '', tokenBudget: 0 };
    }

    // 构建注入文本
    const summaryParts: string[] = [];
    let usedTokens = 0;

    // 加载谱系链上的所有摘要
    const chain = await this.lineageGraph.traceToRoot(best.session.id);
    for (const ancestor of chain) {
      const summary = await this.store.getLatestSummary(ancestor.id);
      if (summary && usedTokens + summary.tokenCount <= recoveryBudgetTokens) {
        summaryParts.push(
          `--- Session ${ancestor.id} (${ancestor.surface}, ${ancestor.createdAt}) ---\n` +
          summary.summaryText,
        );
        usedTokens += summary.tokenCount;
      }
    }

    const injectedSummary = summaryParts.length > 0
      ? `<session-history>\n${summaryParts.join('\n\n')}\n</session-history>`
      : '';

    return {
      candidates,
      selectedSession: best.session,
      injectedSummary,
      tokenBudget: recoveryBudgetTokens - usedTokens,
    };
  }
}
```

## 2. 五级压缩引擎

### 2.1 CompressionEngine 分层架构

重构现有的 `ContextCompressor`（149行），升级为分层的 `CompressionEngine`。原有类保留为 Level 2 的内部实现。

```
┌─────────────────────────────────────────────────────────┐
│                   CompressionEngine                      │
│                                                          │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────────┐  │
│  │ Level 1  │ │ Level 2  │ │ Level 3  │ │ Level 4   │  │
│  │ 微压缩   │→│ Turn     │→│ Session  │→│ Workspace │  │
│  │ (零LLM)  │ │ Summary  │ │ Summary  │ │ Digest    │  │
│  └──────────┘ └──────────┘ └──────────┘ └───────────┘  │
│                                          ┌───────────┐  │
│                                          │ Level 5   │  │
│                                          │ Lineage   │  │
│                                          │ Handoff   │  │
│                                          └───────────┘  │
│                                                          │
│  ┌──────────────────────────────────────────────────┐   │
│  │           ContextWindowManager                    │   │

│  └──────────────────────────────────────────────────┘   │
```

```typescript
// packages/agent-core/src/compression-engine.ts

export interface CompressionEngineConfig {
  // 上下文窗口
  readonly contextWindowTokens: number;     // 模型上下文窗口大小，如 128_000
  readonly outputReserveTokens: number;     // 为输出预留的 token，如 4_096

  // 各级阈值（占可用 token 的百分比）
  readonly microCompressThreshold: number;  // 默认 0.60 — 60% 时触发微压缩
  readonly turnSummaryThreshold: number;    // 默认 0.70 — 70% 时触发 turn summary
  readonly sessionSummaryThreshold: number; // 默认 0.80 — 80% 时触发 session summary
  readonly workspaceDigestThreshold: number;// 默认 0.90 — 90% 时触发 workspace digest

  // 告警阈值
  readonly warningThreshold: number;        // 默认 0.70
  readonly errorThreshold: number;          // 默认 0.85
  readonly autoCompactThreshold: number;    // 默认 0.90
  readonly blockingThreshold: number;       // 默认 0.95

  // 恢复预算
  readonly recoveryBudgetTokens: number;    // 默认 50_000

  // Claude Code 参考值
  readonly autocompactBufferTokens: number; // 默认 13_000
  readonly postCompactTokenBudget: number;  // 默认 50_000
}

const DEFAULT_ENGINE_CONFIG: CompressionEngineConfig = {
  contextWindowTokens: 128_000,
  outputReserveTokens: 4_096,
  microCompressThreshold: 0.60,
  turnSummaryThreshold: 0.70,
  sessionSummaryThreshold: 0.80,
  workspaceDigestThreshold: 0.90,
  warningThreshold: 0.70,
  errorThreshold: 0.85,
  autoCompactThreshold: 0.90,
  blockingThreshold: 0.95,
  recoveryBudgetTokens: 50_000,
  autocompactBufferTokens: 13_000,
  postCompactTokenBudget: 50_000,
};

export type CompressionLevel = 0 | 1 | 2 | 3 | 4 | 5;

export interface CompressionStepResult {
  readonly level: CompressionLevel;
  readonly messages: readonly AgentMessage[];
  readonly tokensBefore: number;
  readonly tokensAfter: number;
  readonly tokensSaved: number;
  readonly summary?: string;
  readonly llmCost: 'zero' | 'low' | 'medium' | 'high';
}

export class CompressionEngine {
  private readonly config: CompressionEngineConfig;
  private readonly tokenCounter: TokenCounter;   // 后面定义
  private readonly sessionStore: SessionStore;

  constructor(
    config: Partial<CompressionEngineConfig>,
    tokenCounter: TokenCounter,
    sessionStore: SessionStore,
  ) {
    this.config = { ...DEFAULT_ENGINE_CONFIG, ...config };
    this.tokenCounter = tokenCounter;
    this.sessionStore = sessionStore;
  }

  /** 可用 token 预算 = 窗口大小 - 输出预留 */
  get availableTokens(): number {
    return this.config.contextWindowTokens - this.config.outputReserveTokens;
  }

  /** 当前使用率 */
  getUsageRatio(messages: readonly AgentMessage[]): number {
    const total = this.tokenCounter.countMessages(messages);
    return total / this.availableTokens;
  }

  /**
   * 自动选择合适的压缩级别并执行。
   * 返回压缩后的消息和执行的步骤记录。
   */
  async autoCompress(
    messages: readonly AgentMessage[],
    summarizer: LLMSummarizer,
  ): Promise<{
    messages: readonly AgentMessage[];
    steps: readonly CompressionStepResult[];
  }> {
    const steps: CompressionStepResult[] = [];
    let current = messages;
    let ratio = this.getUsageRatio(current);

    // Level 1: 微压缩（零 LLM）
    if (ratio > this.config.microCompressThreshold) {
      const result = this.microCompress(current);
      steps.push(result);
      current = result.messages;
      ratio = this.getUsageRatio(current);
    }

    // Level 2: Turn Summary
    if (ratio > this.config.turnSummaryThreshold) {
      const result = await this.turnSummary(current, summarizer);
      steps.push(result);
      current = result.messages;
      ratio = this.getUsageRatio(current);
    }

    // Level 3: Session Summary
    if (ratio > this.config.sessionSummaryThreshold) {
      const result = await this.sessionSummary(current, summarizer);
      steps.push(result);
      current = result.messages;
      ratio = this.getUsageRatio(current);
    }

    // Level 4: Workspace Digest（极端情况）
    if (ratio > this.config.workspaceDigestThreshold) {
      const result = await this.workspaceDigest(current, summarizer);
      steps.push(result);
      current = result.messages;
    }

    return { messages: current, steps };
  }

  // ... 各级压缩方法详见下文
}
```

### 2.2 Level 1：微压缩（零 LLM）

微压缩不调用 LLM，纯规则驱动，零成本。适合在每轮对话后执行。

**触发条件**：token 使用率 > 60%
**LLM 成本**：零

```typescript
/**
 * 微压缩规则：
 * 1. 工具结果裁剪：根据工具类型差异化截断
 * 2. 图片/文档 base64 剥离：替换为占位符
 * 3. 重复系统消息合并
 * 4. 空消息清理
 */
microCompress(messages: readonly AgentMessage[]): CompressionStepResult {
  const tokensBefore = this.tokenCounter.countMessages(messages);
  let result = [...messages];

  // 规则 1: 差异化工具结果裁剪
  result = result.map(m => {
    if (m.role !== 'tool') return m;
    return { ...m, content: this.pruneToolResult(m) };
  });

  // 规则 2: 剥离 base64 内容
  result = result.map(m => ({
    ...m,
    content: m.content
      .replace(/data:[^;]+;base64,[A-Za-z0-9+/=]{100,}/g,
               '[base64-content-stripped]')
      .replace(/```[\s\S]{5000,}?```/g,
               '```\n[large-code-block-stripped, see tool result]\n```'),
  }));

  // 规则 3: 清理空消息
  result = result.filter(m => m.content.trim().length > 0 || m.toolCalls?.length);

  const tokensAfter = this.tokenCounter.countMessages(result);
  return {
    level: 1,
    messages: result,
    tokensBefore,
    tokensAfter,
    tokensSaved: tokensBefore - tokensAfter,
    llmCost: 'zero',
  };
}
```

**工具结果裁剪规则**：

```typescript
/** 不同工具类型的保留策略 */
private static readonly TOOL_PRUNE_RULES: Record<string, {
  maxChars: number;
  keepStrategy: 'head' | 'tail' | 'head-tail' | 'full';
}> = {
  // 搜索类工具：保留前 N 条结果
  'search-workspace': { maxChars: 1000, keepStrategy: 'head' },
  'list-objects':     { maxChars: 800,  keepStrategy: 'head' },
  'list-tasks':       { maxChars: 800,  keepStrategy: 'head' },

  // 读取类工具：保留首尾（摘要+结论）
  'read-object':      { maxChars: 2000, keepStrategy: 'head-tail' },
  'extract-text':     { maxChars: 1500, keepStrategy: 'head-tail' },

  // 写入类工具：只保留结果摘要（操作已完成，不需要完整输出）
  'create-task':      { maxChars: 200,  keepStrategy: 'head' },
  'update-task':      { maxChars: 200,  keepStrategy: 'head' },
  'create-object':    { maxChars: 200,  keepStrategy: 'head' },
  'update-object':    { maxChars: 200,  keepStrategy: 'head' },

  // 默认规则
  '_default':         { maxChars: 500,  keepStrategy: 'head' },
};

private pruneToolResult(message: AgentMessage): string {
  const toolName = message.metadata?.toolName as string ?? '_default';
  const rule = CompressionEngine.TOOL_PRUNE_RULES[toolName]
    ?? CompressionEngine.TOOL_PRUNE_RULES['_default'];

  const content = message.content;
  if (content.length <= rule.maxChars) return content;

  switch (rule.keepStrategy) {
    case 'head':
      return content.slice(0, rule.maxChars) + '\n… [truncated]';
    case 'tail':
      return '… [truncated]\n' + content.slice(-rule.maxChars);
    case 'head-tail': {
      const half = Math.floor(rule.maxChars / 2);
      return content.slice(0, half) + '\n… [middle truncated] …\n' + content.slice(-half);
    }
    case 'full':
      return content;
  }
}
```

### 2.3 Level 2：Turn Summary（单轮压缩）

**触发条件**：微压缩后 token 使用率仍 > 70%
**LLM 成本**：低（每轮约 500 token 输入 + 100 token 输出）
**策略**：将多轮连续的工具调用-工具结果对压缩为单条摘要

```typescript
/**
 * Turn Summary：
 * 找到连续的 assistant(tool_calls) → tool(result) → assistant(tool_calls) → tool(result) 序列，
 * 将整个序列压缩为一条 assistant 摘要消息。
 * 保护最近 N 轮不压缩。
 */
async turnSummary(
  messages: readonly AgentMessage[],
  summarizer: LLMSummarizer,
): Promise<CompressionStepResult> {
  const tokensBefore = this.tokenCounter.countMessages(messages);
  const PROTECT_LAST_TURNS = 4;  // 保护最近 4 轮

  // 1. 识别可压缩的 turn 块
  const turnBlocks = this.identifyTurnBlocks(messages);

  // 2. 保护最后 N 个 turn block
  const compressible = turnBlocks.slice(0, -PROTECT_LAST_TURNS);
  if (compressible.length === 0) {
    return { level: 2, messages, tokensBefore, tokensAfter: tokensBefore,
             tokensSaved: 0, llmCost: 'low' };
  }

  // 3. 逐块压缩
  const compressed: AgentMessage[] = [];
  let i = 0;

  for (const block of turnBlocks) {
    // 添加 block 之前的非 block 消息
    while (i < block.startIdx) {
      compressed.push(messages[i]);
      i++;
    }

    if (compressible.includes(block)) {
      // 压缩这个 block
      const blockMessages = messages.slice(block.startIdx, block.endIdx + 1);
      const blockText = blockMessages
        .map(m => `[${m.role}${m.toolCalls ? '→' + m.toolCalls.map(tc => tc.name).join(',') : ''}] ${m.content.slice(0, 300)}`)
        .join('\n');

      const summary = await summarizer(TURN_SUMMARY_PROMPT(blockText));
      compressed.push({
        id: generateId('turnsum'),
        role: 'assistant',
        content: `[Turn Summary] ${summary}`,
        timestamp: blockMessages[0].timestamp,
        metadata: { compressed: true, originalTurnCount: blockMessages.length },
      });
    } else {
      // 保护区域，原样保留
      for (let j = block.startIdx; j <= block.endIdx; j++) {
        compressed.push(messages[j]);
      }
    }
    i = block.endIdx + 1;
  }

  // 添加剩余消息
  while (i < messages.length) {
    compressed.push(messages[i]);
    i++;
  }

  const tokensAfter = this.tokenCounter.countMessages(compressed);
  return {
    level: 2, messages: compressed, tokensBefore, tokensAfter,
    tokensSaved: tokensBefore - tokensAfter, llmCost: 'low',
  };
}
```

```typescript
async sessionSummary(
  messages: readonly AgentMessage[],
  summarizer: LLMSummarizer,
): Promise<CompressionStepResult> {
  const tokensBefore = this.tokenCounter.countMessages(messages);

  // 保护头部（system prompt + 首轮交互）和尾部（最近对话）
  const HEAD_PROTECT = 2;
  const TAIL_PROTECT_TOKENS = 8_000;

  const head = messages.slice(0, HEAD_PROTECT);
  const { tailStart } = this.findTailBoundary(messages, HEAD_PROTECT, TAIL_PROTECT_TOKENS);
  const tail = messages.slice(tailStart);
  const middle = messages.slice(HEAD_PROTECT, tailStart);

  if (middle.length === 0) {
    return { level: 3, messages, tokensBefore, tokensAfter: tokensBefore,
             tokensSaved: 0, llmCost: 'medium' };
  }

  // 构建中间段文本
  const middleText = middle
    .map(m => `[${m.role}] ${m.content.slice(0, 500)}`)
    .join('\n');

  const summaryJson = await summarizer(SESSION_SUMMARY_PROMPT(middleText));

  // 解析为 11 维结构
  let dimensions: SessionSummaryDimensions;
  try {
    dimensions = JSON.parse(summaryJson);
  } catch {
    dimensions = { request: summaryJson } as SessionSummaryDimensions;
  }

  // 构建紧凑的摘要消息
  const summaryText = Object.entries(dimensions)
    .filter(([, v]) => v && v.trim())
    .map(([k, v]) => `**${k}**: ${v}`)
    .join('\n');

  const summaryMessage: AgentMessage = {
    id: generateId('sessum'),
    role: 'system',
    content: `[Session Summary — 11-Dimension Analysis]\n${summaryText}`,
    timestamp: new Date().toISOString(),
    metadata: { compressed: true, level: 3, dimensions },
  };

  const compressed = [...head, summaryMessage, ...tail];
  const tokensAfter = this.tokenCounter.countMessages(compressed);

  // 持久化摘要
  await this.sessionStore.saveSummary({
    id: generateId('sum'),
    sessionId: messages[0]?.metadata?.sessionId as string ?? '',
    level: 3,
    summaryText,
    dimensionsJson: JSON.stringify(dimensions),
    tokenCount: this.tokenCounter.countText(summaryText),
    sourceRange: `${HEAD_PROTECT}:${tailStart}`,
    createdAt: new Date().toISOString(),
  });

  return {
    level: 3, messages: compressed, tokensBefore, tokensAfter,
    tokensSaved: tokensBefore - tokensAfter, summary: summaryText,
    llmCost: 'medium',
  };
}
```

### 2.6 Level 5：Lineage Handoff

**触发条件**：workspace digest 后仍不够，或会话自然结束需要传承
**LLM 成本**：与 Session Summary 相当
**策略**：结束当前会话，创建全新会话，注入结构化摘要 + 恢复预算

```typescript
/**
 * Lineage Handoff 流程：
 * 1. 对当前会话执行 Session Summary（若未执行）
 * 2. 标记当前会话为 completed，end_reason = 'handoff'
 * 3. 创建新会话，lineage = [{ type: 'compressed_into', sourceId: 旧会话ID }]
 * 4. 在新会话中注入：Session Summary + Recovery Budget 回读内容
 * 5. 分配 50K token 恢复预算
 */
async lineageHandoff(
  currentSession: PersistedSession,
  messages: readonly AgentMessage[],
  summarizer: LLMSummarizer,
): Promise<{
  newSession: PersistedSession;
  injectedMessages: readonly AgentMessage[];
}> {
  // 1. 确保有 Session Summary
  let summary = await this.sessionStore.getLatestSummary(currentSession.id);
  if (!summary || summary.level < 3) {
    const result = await this.sessionSummary(messages, summarizer);
    summary = await this.sessionStore.getLatestSummary(currentSession.id);
  }

  // 2. 标记旧会话
  await this.sessionStore.update(currentSession.id, {
    status: 'completed',
    endReason: 'handoff',
  });

  // 3. 恢复预算分配
  const recoveryBudget = this.config.recoveryBudgetTokens; // 50_000
  const recoveryContent = await this.allocateRecoveryBudget(
    currentSession, summary, recoveryBudget,
  );

  // 4. 构建新会话的初始消息
  const systemPrompt = messages.find(m => m.role === 'system');
  const injectedMessages: AgentMessage[] = [];

  if (systemPrompt) {
    injectedMessages.push(systemPrompt);
  }

  // 注入谱系摘要
  injectedMessages.push({
    id: generateId('handoff'),
    role: 'system',
    content: [
      '[Lineage Handoff — Continuing from previous session]',
      '',
      '## Session History',
      summary?.summaryText ?? '(no summary available)',
      '',
      '## Recovery Context',
      recoveryContent,
    ].join('\n'),
    timestamp: new Date().toISOString(),
    metadata: { compressed: true, level: 5, sourceSessionId: currentSession.id },
  });

  // 5. 创建新会话
  const newSession: PersistedSession = {
    id: generateId('ses'),
    workspaceId: currentSession.workspaceId,
    surface: currentSession.surface,
    anchorObjectIds: currentSession.anchorObjectIds,
    lineage: [{ type: 'compressed_into', sourceId: currentSession.id }],
    status: 'active',
    messages: injectedMessages,
    tokenCount: this.tokenCounter.countMessages(injectedMessages),
    compressionLevel: 5,
    createdAt: new Date().toISOString(),
    updatedAt: new Date().toISOString(),
  };

  await this.sessionStore.save(newSession);
  await this.sessionStore.addLineage(newSession.id, {
    type: 'compressed_into',
    sourceId: currentSession.id,
    metadata: { recoveryBudget, summaryTokens: summary?.tokenCount ?? 0 },
  });

  return { newSession, injectedMessages };
}
```

**恢复预算分配算法**：

```typescript
/**
 * 恢复预算分配优先级：
 * 1. 关键对象上下文（anchor objects）— 40% 预算
 * 2. 未完成任务列表 — 20% 预算
 * 3. 最近 N 轮完整对话（从尾部恢复）— 30% 预算
 * 4. 谱系链上的历史摘要 — 10% 预算
 */
private async allocateRecoveryBudget(
  session: PersistedSession,
  summary: SessionSummaryRecord | null,
  totalBudget: number,
): Promise<string> {
  const parts: string[] = [];
  let usedTokens = 0;

  const ANCHOR_BUDGET = Math.floor(totalBudget * 0.4);
  const TASK_BUDGET = Math.floor(totalBudget * 0.2);
  const RECENT_BUDGET = Math.floor(totalBudget * 0.3);
  const HISTORY_BUDGET = Math.floor(totalBudget * 0.1);

  // 1. Anchor objects
  for (const objectId of session.anchorObjectIds) {
    if (usedTokens >= ANCHOR_BUDGET) break;
    // 从对象图加载对象及其一跳邻居的摘要
    const objectContext = `[Object: ${objectId}] — loaded from object graph`;
    const tokens = this.tokenCounter.countText(objectContext);
    if (usedTokens + tokens <= ANCHOR_BUDGET) {
      parts.push(objectContext);
      usedTokens += tokens;
    }
  }

  // 2. 未完成任务
  // 从 11 维摘要的 openQuestions 和 nextStep 提取
  if (summary?.dimensionsJson) {
    try {
      const dims = JSON.parse(summary.dimensionsJson) as SessionSummaryDimensions;
      const taskContext = [
        dims.openQuestions ? `Open questions: ${dims.openQuestions}` : '',
        dims.nextStep ? `Next step: ${dims.nextStep}` : '',
        dims.artifacts ? `Artifacts: ${dims.artifacts}` : '',
      ].filter(Boolean).join('\n');

      const tokens = this.tokenCounter.countText(taskContext);
      if (tokens <= TASK_BUDGET) {
        parts.push(taskContext);
        usedTokens += tokens;
      }
    } catch { /* ignore parse errors */ }
  }

  // 3. 最近对话（在 handoff 前从原始消息恢复最后几轮）
  // 这部分由调用者在构建新会话时处理

  // 4. 历史摘要（沿谱系链加载）
  // 由 SessionResumptionService.resume() 处理

  return parts.join('\n\n');
}
```

```typescript
export class ContextWindowManager {
  private readonly config: CompressionEngineConfig;
  private readonly tokenCounter: TokenCounter;
  private readonly compressionEngine: CompressionEngine;
  private alerts: ContextWindowAlert[] = [];

  constructor(
    config: CompressionEngineConfig,
    tokenCounter: TokenCounter,
    compressionEngine: CompressionEngine,
  ) {

    this.tokenCounter = tokenCounter;
    this.compressionEngine = compressionEngine;
  }

   * 评估当前上下文状态并返回告警。
   * 每次 LLM 调用前都应检查。

    const usageRatio = currentTokens / availableTokens;
    const compressionLevel = this.detectCompressionLevel(messages);

    this.alerts = [];

      alertLevel = 'warning';
      this.alerts.push({
        level: 'warning',
        usageRatio,
        currentTokens,
        availableTokens,
        message: `Context window at ${(usageRatio * 100).toFixed(1)}%. ` +
                 `Consider wrapping up or the system will auto-compress.`,
        action: 'notify',
      });
    }

    // Level 2: Error（85%）
    if (usageRatio >= this.config.errorThreshold) {
      alertLevel = 'error';
      this.alerts.push({
        level: 'error',
        usageRatio,
        currentTokens,
        availableTokens,
        message: `Context window critically full at ${(usageRatio * 100).toFixed(1)}%. ` +
                 `Auto-compression imminent.`,
        action: 'notify',
      });
    }

    // Level 3: AutoCompact（90%）
    if (usageRatio >= this.config.autoCompactThreshold) {
      alertLevel = 'autoCompact';
      this.alerts.push({
        level: 'autoCompact',
        usageRatio,
        currentTokens,
        availableTokens,
        message: `Auto-compacting context. Token usage: ${currentTokens}/${availableTokens}.`,
        action: 'compress',
      });
    }

    // Level 4: Blocking（95%）
    if (usageRatio >= this.config.blockingThreshold) {
      alertLevel = 'blocking';
      this.alerts.push({
        level: 'blocking',
        usageRatio,
        currentTokens,
        availableTokens,
        message: `Context window FULL. Cannot proceed without compression or handoff.`,
        action: 'block',
      });
    }

    return { currentTokens, availableTokens, usageRatio, alertLevel, compressionLevel, alerts: this.alerts };
  }

  /**
   * 在 Orchestrator 的 agentic loop 中每轮调用。
   * 根据告警级别自动执行压缩或阻断。
   */
  async enforcePolicy(
    messages: readonly AgentMessage[],
    summarizer: LLMSummarizer,
    session: PersistedSession,
  ): Promise<{
    messages: readonly AgentMessage[];
    action: 'continue' | 'compressed' | 'handoff' | 'blocked';
  }> {
    const state = this.evaluate(messages);

    switch (state.alertLevel) {
      case 'normal':
      case 'warning':
        // 正常继续，warning 只发通知
        return { messages, action: 'continue' };

      case 'error':
        // 发出强烈警告但仍继续
        return { messages, action: 'continue' };

      case 'autoCompact': {
        // 自动压缩
        const result = await this.compressionEngine.autoCompress(messages, summarizer);
        return { messages: result.messages, action: 'compressed' };
      }

      case 'blocking': {
        // 先尝试自动压缩
        const result = await this.compressionEngine.autoCompress(messages, summarizer);
        const afterState = this.evaluate(result.messages);

        if (afterState.alertLevel === 'blocking') {
          // 压缩后仍然满，执行 lineage handoff
          return { messages: result.messages, action: 'handoff' };
        }
        return { messages: result.messages, action: 'compressed' };
      }
    }
  }

  private detectCompressionLevel(messages: readonly AgentMessage[]): CompressionLevel {
    for (const m of messages) {
      if (m.metadata?.level === 5) return 5;
      if (m.metadata?.level === 4) return 4;
      if (m.metadata?.level === 3) return 3;
      if (m.metadata?.compressed && m.content.startsWith('[Turn Summary]')) return 2;
    }
    return 0;
  }
}
```

### 3.3 混合 Token 计算

参考 Claude Code 的 `tokenCountWithEstimation()` 实现混合策略：优先使用 API 返回的精确值，回退到 chars/4 估算，并针对中文做修正。

```typescript
// packages/agent-core/src/token-counter.ts

export interface TokenCountCache {
  readonly promptTokens: number;   // API 返回的精确值
  readonly timestamp: number;
  readonly messageCount: number;
}

export interface TokenCounter {
  /** 计算单条文本的 token 数 */
  countText(text: string): number;

  /** 计算消息列表的 token 总数 */
  countMessages(messages: readonly AgentMessage[]): number;

  /** 更新 API 精确值缓存 */
  updateFromApiUsage(usage: TokenUsage, messageCount: number): void;
}

export class HybridTokenCounter implements TokenCounter {
  private apiCache: TokenCountCache | null = null;

  // 每个字符的 token 系数（经验值）
  private static readonly CHAR_TOKEN_RATIOS = {
    ascii: 0.25,          // 英文约 4 字符 = 1 token
    cjk: 0.6,             // 中文约 1.5-2 字符 = 1 token（tiktoken cl100k_base 编码下）
    mixed: 0.35,          // 混合文本取中间值
  };

  /**
   * 混合 token 估算：
   * 1. 如果有 API 缓存且消息数变化 <= 2，用缓存值 + 增量估算
   * 2. 否则用字符级别估算，区分中英文
   */
  countText(text: string): number {
    // 统计 CJK 字符占比
    const cjkChars = (text.match(/[\u4e00-\u9fff\u3400-\u4dbf\uf900-\ufaff]/g) ?? []).length;
    const totalChars = text.length;

    if (totalChars === 0) return 0;

    const cjkRatio = cjkChars / totalChars;

    if (cjkRatio > 0.5) {
      // 中文为主
      return Math.ceil(
        cjkChars * HybridTokenCounter.CHAR_TOKEN_RATIOS.cjk +
        (totalChars - cjkChars) * HybridTokenCounter.CHAR_TOKEN_RATIOS.ascii
      );
    } else if (cjkRatio > 0.1) {
      // 混合文本
      return Math.ceil(totalChars * HybridTokenCounter.CHAR_TOKEN_RATIOS.mixed);
    } else {
      // 英文为主
      return Math.ceil(totalChars * HybridTokenCounter.CHAR_TOKEN_RATIOS.ascii);
    }
  }

  countMessages(messages: readonly AgentMessage[]): number {
    // 如果有新鲜的 API 缓存（消息数差异 <= 2），使用增量策略
    if (this.apiCache && Math.abs(messages.length - this.apiCache.messageCount) <= 2) {
      const newMessages = messages.slice(this.apiCache.messageCount);
      const delta = newMessages.reduce((sum, m) => sum + this.countText(m.content), 0);
      // 每条消息有 ~4 token 的 overhead（role, separators）
      const overhead = newMessages.length * 4;
      return this.apiCache.promptTokens + delta + overhead;
    }

    // 完全估算
    let total = 0;
    for (const m of messages) {
      total += this.countText(m.content);
      total += 4; // message overhead
      if (m.toolCalls) {
        for (const tc of m.toolCalls) {
          total += this.countText(tc.name) + this.countText(tc.arguments);
          total += 10; // tool call overhead
        }
      }
    }
    total += 3; // conversation overhead
    return total;
  }

  updateFromApiUsage(usage: TokenUsage, messageCount: number): void {
    this.apiCache = {
      promptTokens: usage.promptTokens,
      timestamp: Date.now(),
      messageCount,
    };
  }
}
```

### 3.4 PTL（Prompt Too Long）自动降级策略

当 API 返回 "prompt too long" 错误时的紧急处理，参考 Claude Code 的 PTL 重试截断策略。

```typescript
// packages/agent-core/src/ptl-handler.ts

export interface PTLStrategy {
  readonly name: string;
  readonly execute: (messages: readonly AgentMessage[]) => readonly AgentMessage[];
}

export class PTLHandler {
  private readonly strategies: PTLStrategy[] = [
    {
      // 策略 1: 丢弃头部 20%（参考 Claude Code）
      name: 'drop-head-20',
      execute: (messages) => {
        const systemMsgs = messages.filter(m => m.role === 'system');
        const nonSystem = messages.filter(m => m.role !== 'system');
        const dropCount = Math.ceil(nonSystem.length * 0.2);
        return [...systemMsgs, ...nonSystem.slice(dropCount)];
      },
    },
    {
      // 策略 2: 截断所有工具结果到 100 字符
      name: 'truncate-tools-aggressive',
      execute: (messages) => messages.map(m =>
        m.role === 'tool'
          ? { ...m, content: m.content.slice(0, 100) + ' [PTL-truncated]' }
          : m
      ),
    },
    {
      // 策略 3: 只保留 system + 最后 5 条消息
      name: 'emergency-tail-only',
      execute: (messages) => {
        const system = messages.filter(m => m.role === 'system');
        const rest = messages.filter(m => m.role !== 'system');
        return [...system, ...rest.slice(-5)];
      },
    },
  ];

  /**
   * 逐级尝试降级策略，直到 token 低于限制。
   */
  async handlePTL(
    messages: readonly AgentMessage[],
    tokenCounter: TokenCounter,
    contextLimit: number,
  ): Promise<readonly AgentMessage[]> {
    let current = messages;

    for (const strategy of this.strategies) {
      current = strategy.execute(current);
      const tokens = tokenCounter.countMessages(current);
      if (tokens < contextLimit * 0.85) {
        // 成功降到安全水位
        return current;
      }
    }

    // 所有策略都失败，返回最小上下文
    return current;
  }
}
```

### 3.5 恢复预算实现

恢复预算确保压缩后仍有足够空间回读关键对象。这是压缩设计中"预算守恒"原则的核心。

```typescript
/**
 * 压缩后的 token 分配模型：
 *
 * ┌──────────────────────────────────────────────────────┐
 * │              可用 Token 空间 (123,904)               │
 * ├──────────────────────────────────────────────────────┤
 * │  System Prompt  │  Compressed History  │  Recovery   │
 * │    (~2,000)     │    (flexible)        │  Budget     │
 * │                 │                      │  (50,000)   │
 * ├──────────────────────────────────────────────────────┤
 * │                 │  New Conversation Space             │
 * │                 │  (= Available - Prompt - Recovery)  │
 * └──────────────────────────────────────────────────────┘
 *
 * POST_COMPACT_TOKEN_BUDGET = 50,000：压缩后至少留 50K 给新对话
 * AUTOCOMPACT_BUFFER_TOKENS = 13,000：触发压缩时的缓冲区
 */

export interface RecoveryBudgetAllocation {
  readonly anchorObjectTokens: number;    // 40%: 关键对象
  readonly pendingTaskTokens: number;     // 20%: 未完成任务
  readonly recentConvoTokens: number;     // 30%: 最近对话
  readonly historyTokens: number;         // 10%: 谱系历史
  readonly totalAllocated: number;
}

export function computeRecoveryBudget(
  totalRecoveryTokens: number,
  anchorObjectCount: number,
): RecoveryBudgetAllocation {
  return {
    anchorObjectTokens: Math.floor(totalRecoveryTokens * 0.4),
    pendingTaskTokens: Math.floor(totalRecoveryTokens * 0.2),
    recentConvoTokens: Math.floor(totalRecoveryTokens * 0.3),
    historyTokens: Math.floor(totalRecoveryTokens * 0.1),
    totalAllocated: totalRecoveryTokens,
  };
}
```

压缩质量是 M5 的关键风险点。需要专门的集成测试验证摘要保留关键信息。

```typescript
// compression-quality.integration.test.ts

describe('Compression Quality', () => {
  // 准备：构建一个包含已知关键信息的长对话
  const KNOWN_FACTS = [
    { type: 'object_id', value: 'task:task-123', description: '被修改的任务' },

    { type: 'error', value: 'ECONNREFUSED', description: '遇到的网络错误' },

    }

    expect(countPreservedFacts(l1.messages, KNOWN_FACTS))
```

## 附录 C：迁移策略

现有 `ContextCompressor` 类将被标记为 `@deprecated`，但不立即删除：

```typescript

 * Retained for backward compatibility during M5 migration.
 * Will be removed in M6.
 */
export class ContextCompressor {
  // ... 现有 149 行代码保持不变 ...
}
```

`Orchestrator.assembleContext()` 将从直接调用 `ContextCompressor.compress()` 改为调用 `ContextWindowManager.enforcePolicy()`，后者内部使用 `CompressionEngine`。这保证了 Orchestrator 的公开接口不变，只是内部实现升级。
