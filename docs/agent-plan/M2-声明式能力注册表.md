## 1. Capability 接口完整设计

### 1.1 核心问题

当前 `ToolDefinition`（`types.ts:195-205`）的所有治理字段都是**静态常量**：

```typescript
// 当前：静态声明，无法区分同一能力在不同输入下的行为差异
interface ToolDefinition {
  readonly riskLevel: RiskLevel;       // 写死为 R0-read 或 R2-external-read
  readonly approvalPolicy: ApprovalPolicy; // 写死为 A0-auto 或 A2-confirm
  // ...
}
```

这导致一个根本性缺陷：**同一个能力（如 `research.search.external`）在搜索公开 Wikipedia 和搜索付费 IEEE 数据库时，共享相同的风险等级和审批策略**。Claude Code 通过 `isConcurrencySafe(input)` 等输入敏感方法解决了此问题，Orbit 需要借鉴并扩展这一模式。

### 1.2 Capability<Input, Output> 泛型接口

```typescript
// packages/agent-core/src/capability-types.ts

import { z, ZodSchema } from 'zod';
import type {
  AgentDomain,
  RiskLevel,
  ApprovalPolicy,
  ExecutionMode,
  ScopeLimit,
  DataBoundary,
} from './types.js';

// ---- 并发安全等级 ----
export const CONCURRENCY_SAFETY_LEVELS = ['safe', 'path_scoped', 'exclusive'] as const;
export type ConcurrencySafety = (typeof CONCURRENCY_SAFETY_LEVELS)[number];

// ---- 成本档位 ----
export interface CostProfile {
  /** 预估 token 消耗范围 */
  readonly estimatedTokens: { readonly min: number; readonly max: number };
  /** 预估执行时间（毫秒） */
  readonly estimatedDurationMs: { readonly min: number; readonly max: number };
  /** 预估金额（美分），0 = 免费 */
  readonly estimatedCostCents: number;
  /** 是否可能触发外部计费 API */
  readonly externalBilling: boolean;
}

// ---- 执行上下文 ----
export interface ExecutionContext {
  /** 当前 session ID */
  readonly sessionId: string;
  /** 当前 run ID */
  readonly runId: string;
  /** 触发此能力的 Agent 域 */
  readonly agentDomain: AgentDomain;
  /** 工作区 ID */
  readonly workspaceId: string;
  /** 当前对象作用域路径（用于 path_scoped 并发判断） */
  readonly scopePath?: string;
  /** 剩余执行预算 */
  readonly budget: ExecutionBudget;
  /** AbortSignal，用于取消传播 */
  readonly signal: AbortSignal;
}

// ---- 执行预算 ----
export interface ExecutionBudget {
  /** 剩余可用 token 数 */
  readonly remainingTokens: number;
  /** 剩余可用时间（毫秒） */
  readonly remainingTimeMs: number;
  /** 剩余可用金额（美分） */
  readonly remainingCostCents: number;
}

// ---- 能力事件（流式执行产出） ----
export type CapabilityEventKind =
  | 'started'
  | 'progress'
  | 'partial_result'
  | 'budget_warning'
  | 'completed'
  | 'failed';

export interface CapabilityEvent<Output = unknown> {
  readonly kind: CapabilityEventKind;
  readonly capabilityId: string;
  readonly timestamp: string;
  readonly progress?: number;           // 0.0 ~ 1.0
  readonly message?: string;
  readonly partialResult?: Partial<Output>;
  readonly error?: string;
}

// ---- Capability 核心接口 ----
export interface Capability<Input = unknown, Output = unknown> {
  /** 全局唯一 ID，使用点分域命名：`domain.verb.target` */
  readonly id: string;

  /** 所属领域 */
  readonly domain: AgentDomain;

  /** 人类可读描述 */
  readonly description: string;

  /** Zod 输入 schema，用于运行时验证和 LLM tool_call 参数校验 */
  readonly inputSchema: ZodSchema<Input>;

  /** Zod 输出 schema，用于结果验证和类型安全 */
  readonly outputSchema: ZodSchema<Output>;

  // ---- 输入敏感的行为声明（函数，非常量） ----

  /** 根据具体输入评估风险等级 */
  riskLevel(input: Input): RiskLevel;

  /** 根据具体输入决定审批策略 */
  approvalPolicy(input: Input): ApprovalPolicy;

  /** 根据具体输入声明并发安全等级 */
  concurrencySafety(input: Input): ConcurrencySafety;

  // ---- 静态治理元数据 ----

  /** 执行模式 */
  readonly executionMode: ExecutionMode;

  /** 作用域限制 */
  readonly scopeLimit: ScopeLimit;

  /** 成本档案 */
  readonly costProfile: CostProfile;

  /** 数据出境边界 */
  readonly dataBoundary: DataBoundary;

  // ---- 执行 ----

  /**
   * 流式执行能力。
   * 通过 AsyncGenerator yield CapabilityEvent 报告进度，
   * 最终 return Output 返回结果。
   */
  execute(input: Input, context: ExecutionContext): AsyncGenerator<CapabilityEvent<Output>, Output>;
}
```

### 1.3 CostProfile 设计要点

CostProfile 不是精确计费系统，而是**预算守卫的判断依据**。设计原则：

| 字段 | 用途 | 示例 |
|------|------|------|
| `estimatedTokens.min/max` | 上下文预算分配：Executor 在调度能力前检查剩余 token 是否足够容纳预估结果 | `read-object`: {min: 100, max: 5000} |
| `estimatedDurationMs.min/max` | 超时控制：Executor 设置 AbortController 超时 = max × 1.5 | `web-search`: {min: 1000, max: 30000} |
| `estimatedCostCents` | 金额预算告警：累计超过 session 预算阈值时 yield `budget_warning` 事件 | `gpt4-summarize`: 5 |
| `externalBilling` | 审批决策辅助：`externalBilling: true` 的能力在 auto 模式下也需要一次性 session 确认 | `research.search.ieee`: true |

### 1.4 Zod Schema 集成方案

**为什么选择 Zod**：

1. **运行时验证 + 类型推导一体化**：`z.infer<typeof schema>` 自动推导 TypeScript 类型，避免类型定义和验证逻辑的重复
2. **JSON Schema 转换**：`zod-to-json-schema` 可将 Zod schema 转为 JSON Schema，直接作为 OpenAI function calling 的 parameters
3. **生态匹配**：Orbit 的 monorepo 已有 TypeScript 基础设施，Zod 是 TS-first 的 schema 库
4. **错误信息对 LLM 友好**：Zod 的错误消息结构化，可以直接回传给 LLM 要求修正参数

**集成层设计**：

```typescript
// packages/agent-core/src/capability-schema.ts

import { zodToJsonSchema } from 'zod-to-json-schema';
import type { ZodSchema } from 'zod';

/**
 * 将 Capability 的 Zod inputSchema 转为 OpenAI function calling
 * 兼容的 JSON Schema 格式，供 LLM tool_call 使用。
 */
export function capabilityToToolParameters(schema: ZodSchema): Record<string, unknown> {
  const jsonSchema = zodToJsonSchema(schema, { target: 'openAi' });
  // 移除 zod-to-json-schema 添加的顶层 $schema 等非必要字段
  const { $schema, ...rest } = jsonSchema as Record<string, unknown>;
  return rest;
}

/**
 * 验证 LLM 返回的 tool_call arguments，返回类型安全的解析结果。
 * 失败时返回结构化错误信息，可直接回传 LLM 重试。
 */
export function validateCapabilityInput<T>(
  schema: ZodSchema<T>,
  rawArgs: Record<string, unknown>,
): { success: true; data: T } | { success: false; error: string } {
  const result = schema.safeParse(rawArgs);
  if (result.success) {
    return { success: true, data: result.data };
  }
  const errorMessages = result.error.issues
    .map((issue) => `${issue.path.join('.')}: ${issue.message}`)
    .join('; ');
  return { success: false, error: `Input validation failed: ${errorMessages}` };
}
```

**依赖安装**：在 `packages/agent-core/package.json` 中添加：

```json
{
  "dependencies": {
    "zod": "^3.23.0"
  },
  "devDependencies": {
    "zod-to-json-schema": "^3.23.0"
  }
}
```

> **注意**：`zod` 是 runtime 依赖（用于输入验证），`zod-to-json-schema` 是 dev 依赖（仅在构建 LLM tool 定义时使用，可在 capability 注册阶段预转换缓存）。

### 1.5 与现有 ToolDefinition 的关系

`Capability<Input, Output>` 是 `ToolDefinition` 的**超集演进**，两者的映射关系：

| ToolDefinition 字段 | Capability 字段 | 变化 |
|---------------------|----------------|------|
| `name: string` | `id: string` | 命名规则从 `kebab-case` 升级为 `dot.separated.hierarchy` |
| `domain: AgentDomain` | `domain: AgentDomain` | 不变 |
| `description: string` | `description: string` | 不变 |
| `inputSchema: Record<string, unknown>` | `inputSchema: ZodSchema<Input>` | 从松散 JSON → 强类型 Zod |
| — | `outputSchema: ZodSchema<Output>` | 🆕 输出也有 schema |
| `riskLevel: RiskLevel` | `riskLevel(input): RiskLevel` | 静态常量 → 输入敏感函数 |
| `approvalPolicy: ApprovalPolicy` | `approvalPolicy(input): ApprovalPolicy` | 静态常量 → 输入敏感函数 |
| — | `concurrencySafety(input): ConcurrencySafety` | 🆕 |
| `executionMode: ExecutionMode` | `executionMode: ExecutionMode` | 不变 |
| `scopeLimit: ScopeLimit` | `scopeLimit: ScopeLimit` | 不变 |
| — | `costProfile: CostProfile` | 🆕 |
| `dataBoundary: DataBoundary` | `dataBoundary: DataBoundary` | 不变 |
| — | `execute(input, ctx): AsyncGenerator` | 🆕 handler 内聚到接口中 |

**关键设计决策：execute 内聚**。当前 `ToolRegistry` 中 `handler` 是独立注册的函数，与 `ToolDefinition` 分离。新设计将执行逻辑内聚到 `Capability` 接口中——每个 Capability 既声明自己的行为特征，也负责自己的执行。这消除了 definition 与 handler 不匹配的风险。

## 2. 输入敏感行为声明

### 2.1 设计原理

核心洞察来自 Claude Code 的 `Tool<Input, Output, Progress>` 接口：**同一个能力对不同输入应有不同的治理行为**。这不是过度工程——它直接影响用户体验：

- `research.search.external({ source: 'wikipedia' })` → R2-external-read / A0-auto / safe
- `research.search.external({ source: 'ieee', apiKey: 'user-key' })` → R3-external-write / A2-confirm / exclusive

如果用静态声明，要么把 `research.search.external` 统一设为 R3（过度保守，每次搜 Wikipedia 都要确认），要么设为 R2（不够安全，搜付费库时不提醒用户会产生费用）。

### 2.2 三个输入敏感方法的设计

#### riskLevel(input: Input): RiskLevel

```typescript
// 示例：research.search.external 能力
riskLevel(input: SearchExternalInput): RiskLevel {
  // 公开免费源 → 外部读
  if (FREE_SOURCES.includes(input.source)) return 'R2-external-read';
  // 付费 API 源 → 外部写（因为产生计费）
  if (input.apiKey || PAID_SOURCES.includes(input.source)) return 'R3-external-write';
  // 默认 fail-closed
  return 'R3-external-write';
}
```

**设计约束**：
- `riskLevel()` 必须是**纯函数**，不能有副作用、不能访问外部状态
- 返回值必须是 `RISK_LEVELS` 枚举之一
- 函数执行时间必须在 1ms 以内（不能做 IO）

#### approvalPolicy(input: Input): ApprovalPolicy

```typescript
// 示例：ops.delete-object 能力
approvalPolicy(input: DeleteObjectInput): ApprovalPolicy {
  // 删除草稿 → 透明执行
  if (input.objectStatus === 'draft') return 'A1-transparent';
  // 删除已发布内容 → 确认
  if (input.objectStatus === 'published') return 'A2-confirm';
  // 批量删除 → 双重确认
  if (input.objectIds && input.objectIds.length > 5) return 'A3-dual-confirm';
  // 默认 fail-closed
  return 'A2-confirm';
}
```

#### concurrencySafety(input: Input): ConcurrencySafety

```typescript
// 示例：writing.update-object 能力
concurrencySafety(input: UpdateObjectInput): ConcurrencySafety {
  // 元数据更新（标签、分类）→ 路径域安全
  if (input.updateType === 'metadata') return 'path_scoped';
  // 内容更新 → 独占（防止并发写冲突）
  if (input.updateType === 'content') return 'exclusive';
  // 默认 fail-closed
  return 'exclusive';
}
```

**三级含义**：

| 等级 | 含义 | 调度行为 |
|------|------|---------|
| `safe` | 纯只读，无副作用 | 可与任何其他能力并行执行 |
| `path_scoped` | 在特定路径/对象上有副作用 | 同路径串行，不同路径可并行 |
| `exclusive` | 全局副作用 | 必须独占执行，等待所有并行任务完成后再开始 |

### 2.3 Fail-Closed 默认值实现

**方案选择**：使用 `class defaults + 工厂函数` 而非 Proxy。

理由：Proxy 虽然可以自动拦截未定义属性的访问，但在 TypeScript 中类型安全性差、调试困难、性能开销大。工厂函数模式更直观、更可测试。

```typescript
// packages/agent-core/src/capability-defaults.ts

import type { RiskLevel, ApprovalPolicy } from './types.js';
import type {
  Capability,
  ConcurrencySafety,
  CostProfile,
} from './capability-types.js';

/**
 * Fail-closed 默认值常量。
 * 任何未显式声明行为特征的 Capability 都使用这些值。
 */
export const CAPABILITY_DEFAULTS = {
  riskLevel: (): RiskLevel => 'R3-external-write',
  approvalPolicy: (): ApprovalPolicy => 'A2-confirm',
  concurrencySafety: (): ConcurrencySafety => 'exclusive',
  costProfile: {
    estimatedTokens: { min: 0, max: 10000 },
    estimatedDurationMs: { min: 0, max: 30000 },
    estimatedCostCents: 0,
    externalBilling: false,
  } satisfies CostProfile,
} as const;

/**
 * 创建一个安全的 Capability 定义，自动填充未声明的行为方法。
 * 这是注册 Capability 的推荐入口——忘记声明任何行为特征都会
 * 回退到最保守的默认值（fail-closed 原则）。
 *
 * @example
 * ```typescript
 * const readObject = defineCapability({
 *   id: 'reading.read-object',
 *   domain: 'reading',
 *   description: 'Read an object from workspace',
 *   inputSchema: ReadObjectInputSchema,
 *   outputSchema: ReadObjectOutputSchema,
 *   // 显式声明：纯只读，无需审批
 *   riskLevel: () => 'R0-read',
 *   approvalPolicy: () => 'A0-auto',
 *   concurrencySafety: () => 'safe',
 *   executionMode: 'sync',
 *   scopeLimit: 'current-object',
 *   costProfile: { estimatedTokens: { min: 50, max: 2000 }, ... },
 *   dataBoundary: 'local-only',
 *   async *execute(input, ctx) {
 *     yield { kind: 'started', capabilityId: 'reading.read-object', timestamp: now() };
 *     const result = await objectStore.read(input.objectId);
 *     return result;
 *   },
 * });
 * ```
 */
export function defineCapability<Input, Output>(
  partial: Omit<Capability<Input, Output>, 'riskLevel' | 'approvalPolicy' | 'concurrencySafety'>
    & Partial<Pick<Capability<Input, Output>, 'riskLevel' | 'approvalPolicy' | 'concurrencySafety'>>,
): Capability<Input, Output> {
  return {
    riskLevel: CAPABILITY_DEFAULTS.riskLevel as Capability<Input, Output>['riskLevel'],
    approvalPolicy: CAPABILITY_DEFAULTS.approvalPolicy as Capability<Input, Output>['approvalPolicy'],
    concurrencySafety: CAPABILITY_DEFAULTS.concurrencySafety as Capability<Input, Output>['concurrencySafety'],
    ...partial,
  };
}
```

### 2.4 实际示例：research.search.external

完整实现一个具有输入敏感行为声明的能力，展示所有设计要素如何协同：

```typescript
// packages/agent-core/src/capabilities/research-search-external.ts

import { z } from 'zod';
import { defineCapability } from '../capability-defaults.js';
import type { CapabilityEvent } from '../capability-types.js';

// ---- Schema 定义 ----

const FREE_SOURCES = ['wikipedia', 'arxiv', 'semantic-scholar'] as const;
const PAID_SOURCES = ['ieee', 'springer', 'elsevier'] as const;

const SearchExternalInputSchema = z.object({
  query: z.string().min(1).max(500),
  source: z.enum([...FREE_SOURCES, ...PAID_SOURCES]),
  maxResults: z.number().int().min(1).max(50).default(10),
  apiKey: z.string().optional(),
});

const SearchExternalOutputSchema = z.object({
  results: z.array(z.object({
    title: z.string(),
    url: z.string().url(),
    snippet: z.string(),
    relevanceScore: z.number(),
  })),
  totalFound: z.number(),
  source: z.string(),
  cached: z.boolean(),
});

type SearchInput = z.infer<typeof SearchExternalInputSchema>;
type SearchOutput = z.infer<typeof SearchExternalOutputSchema>;

// ---- Capability 定义 ----

export const researchSearchExternal = defineCapability<SearchInput, SearchOutput>({
  id: 'research.search.external',
  domain: 'research',
  description: 'Search external academic and knowledge sources',
  inputSchema: SearchExternalInputSchema,
  outputSchema: SearchExternalOutputSchema,

  // 输入敏感：免费源 vs 付费源
  riskLevel(input) {
    if (FREE_SOURCES.includes(input.source as typeof FREE_SOURCES[number])) {
      return 'R2-external-read';
    }
    return 'R3-external-write'; // 付费源可能产生费用
  },

  approvalPolicy(input) {
    if (FREE_SOURCES.includes(input.source as typeof FREE_SOURCES[number])) {
      return 'A0-auto';
    }
    if (input.apiKey) {
      return 'A1-transparent'; // 用户提供了 API key，信任其意图
    }
    return 'A2-confirm'; // 付费源无 key，需确认是否使用系统额度
  },

  concurrencySafety(_input) {
    return 'safe'; // 搜索是纯只读操作
  },

  executionMode: 'async',
  scopeLimit: 'workspace',
  costProfile: {
    estimatedTokens: { min: 200, max: 5000 },
    estimatedDurationMs: { min: 500, max: 15000 },
    estimatedCostCents: 0, // 由 riskLevel 区分
    externalBilling: true,
  },
  dataBoundary: 'can-egress',

  async *execute(input, context) {
    yield {
      kind: 'started',
      capabilityId: 'research.search.external',
      timestamp: new Date().toISOString(),
      message: `Searching ${input.source} for "${input.query}"`,
    };

    // 预算检查
    if (context.budget.remainingTimeMs < 5000) {
      yield {
        kind: 'budget_warning',
        capabilityId: 'research.search.external',
        timestamp: new Date().toISOString(),
        message: 'Low time budget, results may be incomplete',
      };
    }

    // 模拟搜索执行（实际实现会调用外部 API）
    yield {
      kind: 'progress',
      capabilityId: 'research.search.external',
      timestamp: new Date().toISOString(),
      progress: 0.5,
      message: `Fetching results from ${input.source}...`,
    };

    // ... 实际搜索逻辑 ...

    const result: SearchOutput = {
      results: [],
      totalFound: 0,
      source: input.source,
      cached: false,
    };

    yield {
      kind: 'completed',
      capabilityId: 'research.search.external',
      timestamp: new Date().toISOString(),
      progress: 1.0,
    };

    return result;
  },
});
```

## 3. CapabilityRegistry 实现

### 3.1 完整 API 设计

```typescript
// packages/agent-core/src/capability-registry.ts

import type { ZodSchema } from 'zod';
import type {
  AgentDomain,
  AgentSurface,
  RiskLevel,
  ScopeLimit,
  DataBoundary,
  ApprovalPolicy,
} from './types.js';
import { RISK_LEVELS, SCOPE_LIMITS } from './types.js';
import type {
  Capability,
  ConcurrencySafety,
  ExecutionBudget,
} from './capability-types.js';

// ---- 七维过滤器 ----

export interface CapabilityFilters {
  /** 1. 入口过滤：当前产品入口只能看到相关域的能力 */
  readonly surface?: AgentSurface;
  /** 2. 权限过滤：当前 Agent 域限定的 allowed/blocked 列表 */
  readonly agentDomain?: AgentDomain;
  readonly allowedIds?: readonly string[];
  readonly blockedIds?: readonly string[];
  /** 3. 设备过滤：运行平台（iOS 可能缺少某些能力） */
  readonly platform?: 'web' | 'electron' | 'ios';
  /** 4. 作用域过滤：不超过指定作用域级别 */
  readonly maxScopeLimit?: ScopeLimit;
  /** 5. 隐私过滤：数据出境边界约束 */
  readonly dataBoundary?: DataBoundary;
  /** 6. 审批过滤：排除需要某级审批以上的能力（如在 auto 模式下） */
  readonly maxApprovalPolicy?: ApprovalPolicy;
  /** 7. 预算过滤：排除超过当前预算的能力 */
  readonly budget?: ExecutionBudget;
}

// ---- 能力注册条目 ----

export interface CapabilityEntry<Input = unknown, Output = unknown> {
  readonly capability: Capability<Input, Output>;
  /** 来源标记：内部注册 vs MCP 发现 */
  readonly source: 'internal' | 'mcp';
  /** MCP 服务器标识（仅 source === 'mcp'） */
  readonly mcpServerId?: string;
  /** 运行时可用性检查（连接断开时返回 false） */
  readonly checkAvailability?: () => boolean;
  /** 注册时间戳 */
  readonly registeredAt: string;
}

// ---- CapabilityRegistry ----

export class CapabilityRegistry {
  private readonly capabilities = new Map<string, CapabilityEntry>();

  // ---- 注册 API ----

  /**
   * 注册一个能力。如果 ID 已存在，抛出错误（防止覆盖）。
   */
  register<Input, Output>(
    capability: Capability<Input, Output>,
    options?: { source?: 'internal' | 'mcp'; mcpServerId?: string; checkAvailability?: () => boolean },
  ): void {
    if (this.capabilities.has(capability.id)) {
      throw new Error(`Capability "${capability.id}" is already registered`);
    }
    this.capabilities.set(capability.id, {
      capability: capability as Capability,
      source: options?.source ?? 'internal',
      mcpServerId: options?.mcpServerId,
      checkAvailability: options?.checkAvailability,
      registeredAt: new Date().toISOString(),
    });
  }

  /**
   * 注销一个能力（主要用于 MCP 服务器断开连接时）。
   */
  deregister(id: string): void {
    if (!this.capabilities.has(id)) {
      throw new Error(`Capability "${id}" is not registered`);
    }
    this.capabilities.delete(id);
  }

  /**
   * 批量注册（从 MCP 服务器发现时使用）。
   * 如果某个 ID 已存在，跳过而非抛错（MCP 重连场景）。
   */
  registerBatch(
    entries: ReadonlyArray<{ capability: Capability; mcpServerId: string; checkAvailability?: () => boolean }>,
  ): { registered: number; skipped: number } {
    let registered = 0;
    let skipped = 0;
    for (const entry of entries) {
      if (this.capabilities.has(entry.capability.id)) {
        skipped++;
        continue;
      }
      this.capabilities.set(entry.capability.id, {
        capability: entry.capability,
        source: 'mcp',
        mcpServerId: entry.mcpServerId,
        checkAvailability: entry.checkAvailability,
        registeredAt: new Date().toISOString(),
      });
      registered++;
    }
    return { registered, skipped };
  }

  /**
   * 批量注销某个 MCP 服务器的所有能力。
   */
  deregisterByMcpServer(mcpServerId: string): number {
    let count = 0;
    for (const [id, entry] of this.capabilities) {
      if (entry.mcpServerId === mcpServerId) {
        this.capabilities.delete(id);
        count++;
      }
    }
    return count;
  }

  // ---- 发现 API ----

  has(id: string): boolean {
    return this.capabilities.has(id);
  }

  get<Input = unknown, Output = unknown>(id: string): CapabilityEntry<Input, Output> | undefined {
    return this.capabilities.get(id) as CapabilityEntry<Input, Output> | undefined;
  }

  getAllIds(): readonly string[] {
    return [...this.capabilities.keys()];
  }

  getByDomain(domain: AgentDomain): readonly CapabilityEntry[] {
    return [...this.capabilities.values()].filter(
      (e) => e.capability.domain === domain,
    );
  }

  // ---- 七维过滤 ----

  /**
   * 根据 CapabilityFilters 返回当前上下文下可用的能力列表。
   * 过滤逻辑遵循 fail-closed 原则：任一维度不满足即排除。
   *
   * 七维过滤顺序（从低成本到高成本）：
   * 1. 入口亲和度（surface → domain affinity map）
   * 2. 权限白名单/黑名单（agentDomain → allowed/blocked）
   * 3. 设备能力（platform → capability 兼容性）
   * 4. 作用域限制（scopeLimit 等级比较）
   * 5. 隐私/数据边界（dataBoundary 匹配）
   * 6. 审批等级限制（仅在指定 maxApprovalPolicy 时生效）
   * 7. 预算约束（costProfile vs 剩余预算）
   */
  getCapabilities(filters?: CapabilityFilters): readonly Capability[] {
    let entries = [...this.capabilities.values()];

    // 排除不可用的能力（MCP 连接断开等）
    entries = entries.filter((e) => !e.checkAvailability || e.checkAvailability());

    // 维度 1: 入口亲和度
    if (filters?.surface) {
      const allowedDomains = SURFACE_DOMAIN_AFFINITY[filters.surface];
      entries = entries.filter((e) => allowedDomains.includes(e.capability.domain));
    }

    // 维度 2: Agent 权限
    if (filters?.agentDomain) {
      const d = filters.agentDomain;
      entries = entries.filter((e) => e.capability.domain === d || isAllowedCrossDomain(e.capability.id, d));
    }
    if (filters?.allowedIds) {
      const allowed = new Set(filters.allowedIds);
      entries = entries.filter((e) => allowed.has(e.capability.id));
    }
    if (filters?.blockedIds) {
      const blocked = new Set(filters.blockedIds);
      entries = entries.filter((e) => !blocked.has(e.capability.id));
    }

    // 维度 3: 设备能力（预留，当前所有能力默认全平台可用）
    // if (filters?.platform) { ... }

    // 维度 4: 作用域
    if (filters?.maxScopeLimit) {
      const maxIdx = SCOPE_LIMITS.indexOf(filters.maxScopeLimit);
      entries = entries.filter(
        (e) => SCOPE_LIMITS.indexOf(e.capability.scopeLimit) <= maxIdx,
      );
    }

    // 维度 5: 数据出境
    if (filters?.dataBoundary) {
      const allowed = DATA_BOUNDARY_COMPATIBILITY[filters.dataBoundary];
      entries = entries.filter((e) => allowed.includes(e.capability.dataBoundary));
    }

    // 维度 6: 审批等级（无法对所有输入检查，使用 fail-closed 默认值比较）
    // 注意：这里只能使用默认值做粗筛，实际执行时还会根据具体输入再检查
    if (filters?.maxApprovalPolicy) {
      const maxIdx = APPROVAL_POLICIES_ORDER.indexOf(filters.maxApprovalPolicy);
      entries = entries.filter((e) => {
        // 使用 fail-closed 默认值作为最坏情况估计
        const defaultPolicy = e.capability.approvalPolicy({} as never);
        return APPROVAL_POLICIES_ORDER.indexOf(defaultPolicy) <= maxIdx;
      });
    }

    // 维度 7: 预算约束
    if (filters?.budget) {
      const b = filters.budget;
      entries = entries.filter((e) => {
        const cost = e.capability.costProfile;
        return (
          cost.estimatedTokens.min <= b.remainingTokens &&
          cost.estimatedDurationMs.min <= b.remainingTimeMs &&
          cost.estimatedCostCents <= b.remainingCostCents
        );
      });
    }

    return entries.map((e) => e.capability);
  }

  /** 返回注册表统计信息 */
  stats(): { total: number; internal: number; mcp: number; byDomain: Record<string, number> } {
    let internal = 0;
    let mcp = 0;
    const byDomain: Record<string, number> = {};
    for (const entry of this.capabilities.values()) {
      if (entry.source === 'internal') internal++;
      else mcp++;
      const d = entry.capability.domain;
      byDomain[d] = (byDomain[d] ?? 0) + 1;
    }
    return { total: this.capabilities.size, internal, mcp, byDomain };
  }
}
```

### 3.2 七维过滤逻辑详解

| 维度 | 数据来源 | 判断逻辑 | Fail-Closed 行为 |
|------|---------|---------|-----------------|
| **1. 入口** | `AgentSurface` → `SURFACE_DOMAIN_AFFINITY` 映射表 | 当前入口只看到亲和域的能力，如 `reader` 只看到 `reading` + `research` 域 | 未匹配的域一律排除 |
| **2. 权限** | `DomainAgentConfig.allowedCapabilities` / `blockedCapabilities` | 白名单优先于黑名单；都未命中时 fail-closed 排除 | 未在任何列表中的能力默认不可见 |
| **3. 设备** | 运行时平台检测 | iOS 可能缺少 `electron-only` 的能力（如直接文件系统访问） | 未声明平台兼容性的能力默认全平台可用 |
| **4. 作用域** | `capability.scopeLimit` vs 过滤器 `maxScopeLimit` | 序数比较：`current-object` < `current-space` < `current-project` < `workspace` < `global` | 超出限制的能力一律排除 |
| **5. 隐私** | `capability.dataBoundary` vs 过滤器 `dataBoundary` | `local-only` 上下文中不允许 `can-egress` 能力 | 出境策略不匹配的一律排除 |
| **6. 审批** | 使用空输入调用 `capability.approvalPolicy({})` 获取默认值 | 只在 auto 模式下过滤掉需要审批的能力 | 取默认值（fail-closed 为 A2-confirm）做粗筛 |
| **7. 预算** | `capability.costProfile` vs `ExecutionBudget` | 能力的最低成本超过剩余预算时排除 | 无 CostProfile 时使用默认值（保守估计） |

### 3.4 MCP 动态注册

当 Orbit 作为 MCP Client 连接到外部服务器时，需要将发现的 MCP 工具自动转换为 Capability 并注册。

#### MCP Annotations → Capability 行为声明的映射规则

```typescript
// packages/agent-core/src/mcp-capability-bridge.ts

import { z } from 'zod';
import { defineCapability } from './capability-defaults.js';
import type { Capability, CostProfile } from './capability-types.js';
import type { RiskLevel, ApprovalPolicy, AgentDomain } from './types.js';

/**
 * MCP Tool 的 annotations 结构（来自 MCP 规范）。
 * 参考：https://modelcontextprotocol.io/docs/concepts/tools#annotations
 */
interface McpToolAnnotations {
  readonly readOnlyHint?: boolean;
  readonly destructiveHint?: boolean;
  readonly idempotentHint?: boolean;
  readonly openWorldHint?: boolean;
}

/** MCP 发现的工具定义 */
interface McpToolDefinition {
  readonly name: string;
  readonly description: string;
  readonly inputSchema: Record<string, unknown>;
  readonly annotations?: McpToolAnnotations;
}

/**
 * 将 MCP Tool 转换为 Orbit Capability。
 *
 * 映射规则（fail-closed 原则）：
 *
 * | MCP Annotation        | Orbit 行为                                |
 * |----------------------|-------------------------------------------|
 * | readOnlyHint: true   | riskLevel → R2-external-read              |
 * |                      | approvalPolicy → A0-auto                  |
 * |                      | concurrencySafety → safe                  |
 * | destructiveHint: true| riskLevel → R3-external-write             |
 * |                      | approvalPolicy → A2-confirm               |
 * |                      | concurrencySafety → exclusive             |
 * | 无 annotation        | riskLevel → R3-external-write (fail-closed)|
 * |                      | approvalPolicy → A2-confirm               |
 * |                      | concurrencySafety → exclusive             |
 * | idempotentHint: true | approvalPolicy 可降一级                    |
 * | openWorldHint: true  | dataBoundary → can-egress                 |
 */
export function mcpToolToCapability(
  tool: McpToolDefinition,
  mcpServerId: string,
  domainHint?: AgentDomain,
): Capability {
  const annotations = tool.annotations ?? {};

  // 推导风险等级
  const derivedRiskLevel: RiskLevel = annotations.readOnlyHint
    ? 'R2-external-read'
    : 'R3-external-write'; // fail-closed

  // 推导审批策略
  let derivedApprovalPolicy: ApprovalPolicy = annotations.readOnlyHint
    ? 'A0-auto'
    : 'A2-confirm'; // fail-closed
  // 幂等操作可降一级
  if (annotations.idempotentHint && derivedApprovalPolicy === 'A2-confirm') {
    derivedApprovalPolicy = 'A1-transparent';
  }

  // 推导并发安全
  const derivedConcurrency = annotations.readOnlyHint ? 'safe' as const : 'exclusive' as const;

  // 推导数据边界
  const derivedDataBoundary = annotations.openWorldHint !== false
    ? 'can-egress' as const
    : 'local-only' as const;

  // 用 MCP 的 inputSchema 构建松散 Zod schema（passthrough 模式，允许任意字段）
  const inputSchema = z.record(z.unknown());
  const outputSchema = z.unknown();

  // 构建点分 ID
  const capabilityId = `mcp.${mcpServerId}.${tool.name}`.replace(/[^a-zA-Z0-9._-]/g, '_');

  return defineCapability({
    id: capabilityId,
    domain: domainHint ?? 'ops', // MCP 工具默认归入 ops 域
    description: `[MCP: ${mcpServerId}] ${tool.description}`,
    inputSchema,
    outputSchema,
    riskLevel: () => derivedRiskLevel,
    approvalPolicy: () => derivedApprovalPolicy,
    concurrencySafety: () => derivedConcurrency,
    executionMode: 'async',
    scopeLimit: 'global',
    costProfile: {
      estimatedTokens: { min: 0, max: 10000 },
      estimatedDurationMs: { min: 500, max: 60000 },
      estimatedCostCents: 0,
      externalBilling: true,
    },
    dataBoundary: derivedDataBoundary,
    async *execute(input, context) {
      yield {
        kind: 'started',
        capabilityId,
        timestamp: new Date().toISOString(),
        message: `Calling MCP tool: ${tool.name}`,
      };
      // 实际 MCP 调用会在这里通过 MCP Client SDK 发出
      // 此处为接口占位，具体实现在 MCP 集成模块中
      throw new Error(`MCP execution not implemented: ${tool.name}`);
    },
  });
}

/**
 * 批量转换 MCP 服务器发现的工具列表。
 */
export function mcpToolsToCapabilities(
  tools: readonly McpToolDefinition[],
  mcpServerId: string,
): Capability[] {
  return tools.map((tool) => mcpToolToCapability(tool, mcpServerId));
}
```

## 4. StreamingCapabilityExecutor

### 4.1 并发调度器设计

StreamingCapabilityExecutor 是 Capability 的运行时调度引擎。其核心职责是：根据每个能力的 `concurrencySafety(input)` 声明，智能决定并行/串行执行策略。

设计借鉴 Claude Code 的 `StreamingToolExecutor`，但增加了**混合并发**支持——同一批调用中可以同时并行 `safe` 能力和串行 `exclusive` 能力。

```typescript
// packages/agent-core/src/capability-executor.ts

import type {
  Capability,
  CapabilityEvent,
  ExecutionContext,
  ConcurrencySafety,
} from './capability-types.js';
import type { CapabilityRegistry } from './capability-registry.js';
import { validateCapabilityInput } from './capability-schema.js';

// ---- 调度请求 ----

export interface CapabilityInvocation {
  readonly capabilityId: string;
  readonly input: Record<string, unknown>;
}

// ---- 执行结果 ----

export interface CapabilityExecutionResult {
  readonly capabilityId: string;
  readonly success: boolean;
  readonly output: unknown;
  readonly error?: string;
  readonly durationMs: number;
  readonly tokenEstimate: number;
}

// ---- 预算追踪器 ----

export class BudgetTracker {
  private usedTokens = 0;
  private usedCostCents = 0;
  private readonly startTime: number;

  constructor(
    private readonly budget: {
      maxTokens: number;
      maxTimeMs: number;
      maxCostCents: number;
    },
  ) {
    this.startTime = Date.now();
  }

  get remaining() {
    return {
      remainingTokens: this.budget.maxTokens - this.usedTokens,
      remainingTimeMs: Math.max(0, this.budget.maxTimeMs - (Date.now() - this.startTime)),
      remainingCostCents: this.budget.maxCostCents - this.usedCostCents,
    };
  }

  consume(tokens: number, costCents: number): void {
    this.usedTokens += tokens;
    this.usedCostCents += costCents;
  }

  isExhausted(): boolean {
    const r = this.remaining;
    return r.remainingTokens <= 0 || r.remainingTimeMs <= 0 || r.remainingCostCents < 0;
  }

  /** 退还预算（能力被审批拒绝时） */
  refund(tokens: number, costCents: number): void {
    this.usedTokens = Math.max(0, this.usedTokens - tokens);
    this.usedCostCents = Math.max(0, this.usedCostCents - costCents);
  }
}

// ---- 结果预算控制 ----

const MAX_RESULT_CHARS = 100_000;     // 单个结果最大字符数
const SUMMARY_THRESHOLD = 50_000;     // 超过此阈值触发摘要压缩
const SUMMARY_TARGET_CHARS = 5_000;   // 摘要目标长度

/**
 * 结果预算控制：超长结果自动压缩为摘要+引用。
 * 借鉴 Hermes 的分层结果预算系统。
 */
function constrainResult(output: unknown, capabilityId: string): { output: unknown; compressed: boolean } {
  const serialized = typeof output === 'string' ? output : JSON.stringify(output);
  if (serialized.length <= SUMMARY_THRESHOLD) {
    return { output, compressed: false };
  }
  // 超出阈值：截取前部 + 添加引用标记
  const truncated = serialized.slice(0, SUMMARY_TARGET_CHARS);
  const referenceNote =
    `\n\n[Result truncated: original ${serialized.length} chars → ${SUMMARY_TARGET_CHARS} chars. ` +
    `Full result stored as capability output reference: ${capabilityId}]`;
  return {
    output: truncated + referenceNote,
    compressed: true,
  };
}

// ---- StreamingCapabilityExecutor ----

export class StreamingCapabilityExecutor {
  constructor(
    private readonly registry: CapabilityRegistry,
    private readonly budgetTracker: BudgetTracker,
  ) {}

  /**
   * 批量执行一组能力调用，根据 concurrencySafety 声明智能调度。
   *
   * 调度策略（混合并发）：
   *
   * 1. 将调用分为三组：safe / path_scoped / exclusive
   * 2. safe 组：全部并行执行
   * 3. path_scoped 组：按 scopePath 分桶，同桶串行，不同桶并行
   * 4. exclusive 组：严格串行，每个执行完毕后才启动下一个
   * 5. 三组之间的调度：safe 和 path_scoped 可以与 exclusive 之前的队列并行，
   *    但 exclusive 执行时必须等待所有 safe/path_scoped 完成
   *
   * 通过 AsyncGenerator yield 每个能力的事件流。
   */
  async *executeBatch(
    invocations: readonly CapabilityInvocation[],
    context: ExecutionContext,
  ): AsyncGenerator<CapabilityEvent, readonly CapabilityExecutionResult[]> {
    if (invocations.length === 0) return [];

    // 分类
    const classified = this.classifyInvocations(invocations, context);
    const allResults: CapabilityExecutionResult[] = [];

    // Phase 1: 并行执行 safe + path_scoped（不同路径）
    const parallelTasks: Array<Promise<CapabilityExecutionResult>> = [];

    for (const inv of classified.safe) {
      parallelTasks.push(this.executeSingle(inv, context));
    }

    // path_scoped：按路径分桶
    const pathBuckets = this.groupByPath(classified.pathScoped, context);
    for (const bucket of pathBuckets.values()) {
      // 同桶内串行
      parallelTasks.push(this.executeSequential(bucket, context));
    }

    // 等待所有并行任务完成
    if (parallelTasks.length > 0) {
      const parallelResults = await Promise.all(parallelTasks);
      allResults.push(...parallelResults);
      // 更新预算
      for (const r of parallelResults) {
        this.budgetTracker.consume(r.tokenEstimate, 0);
      }
    }

    // Phase 2: 串行执行 exclusive
    for (const inv of classified.exclusive) {
      if (this.budgetTracker.isExhausted()) {
        allResults.push({
          capabilityId: inv.capabilityId,
          success: false,
          output: '',
          error: 'Execution budget exhausted',
          durationMs: 0,
          tokenEstimate: 0,
        });
        yield {
          kind: 'failed',
          capabilityId: inv.capabilityId,
          timestamp: new Date().toISOString(),
          error: 'Execution budget exhausted',
        };
        continue;
      }
      const result = await this.executeSingle(inv, context);
      this.budgetTracker.consume(result.tokenEstimate, 0);
      allResults.push(result);
    }

    return allResults;
  }

  /**
   * 执行单个能力调用。
   * 内部流程：输入验证 → 预算检查 → 执行 → 结果约束 → 预算更新。
   */
  private async executeSingle(
    invocation: CapabilityInvocation,
    context: ExecutionContext,
  ): Promise<CapabilityExecutionResult> {
    const entry = this.registry.get(invocation.capabilityId);
    if (!entry) {
      return {
        capabilityId: invocation.capabilityId,
        success: false,
        output: '',
        error: `Capability "${invocation.capabilityId}" not found`,
        durationMs: 0,
        tokenEstimate: 0,
      };
    }

    const capability = entry.capability;

    // 1. 输入验证（Zod）
    const validation = validateCapabilityInput(capability.inputSchema, invocation.input);
    if (!validation.success) {
      return {
        capabilityId: invocation.capabilityId,
        success: false,
        output: '',
        error: validation.error,
        durationMs: 0,
        tokenEstimate: 0,
      };
    }

    const validInput = validation.data;
    const start = Date.now();

    // 2. 执行（消费 AsyncGenerator）
    try {
      const generator = capability.execute(validInput, {
        ...context,
        budget: this.budgetTracker.remaining,
      });

      let result: unknown;
      let iterResult = await generator.next();
      while (!iterResult.done) {
        // 每个 yield 的事件可以被上层消费（通过外层 generator 传播）
        iterResult = await generator.next();
      }
      result = iterResult.value;

      // 3. 结果预算控制
      const { output, compressed } = constrainResult(result, invocation.capabilityId);
      const serialized = typeof output === 'string' ? output : JSON.stringify(output);
      const tokenEstimate = Math.ceil(serialized.length / 4);

      return {
        capabilityId: invocation.capabilityId,
        success: true,
        output,
        durationMs: Date.now() - start,
        tokenEstimate,
      };
    } catch (err: unknown) {
      const message = err instanceof Error ? err.message : String(err);
      return {
        capabilityId: invocation.capabilityId,
        success: false,
        output: '',
        error: message,
        durationMs: Date.now() - start,
        tokenEstimate: 0,
      };
    }
  }

  /** 串行执行一组调用 */
  private async executeSequential(
    invocations: readonly CapabilityInvocation[],
    context: ExecutionContext,
  ): Promise<CapabilityExecutionResult> {
    // 返回最后一个结果（简化实现；完整实现应返回数组）
    let lastResult: CapabilityExecutionResult | undefined;
    for (const inv of invocations) {
      lastResult = await this.executeSingle(inv, context);
    }
    return lastResult!;
  }

  /** 根据 concurrencySafety 声明分类 */
  private classifyInvocations(
    invocations: readonly CapabilityInvocation[],
    _context: ExecutionContext,
  ): {
    safe: CapabilityInvocation[];
    pathScoped: CapabilityInvocation[];
    exclusive: CapabilityInvocation[];
  } {
    const safe: CapabilityInvocation[] = [];
    const pathScoped: CapabilityInvocation[] = [];
    const exclusive: CapabilityInvocation[] = [];

    for (const inv of invocations) {
      const entry = this.registry.get(inv.capabilityId);
      if (!entry) {
        exclusive.push(inv); // 未知能力 fail-closed
        continue;
      }
      const safety = entry.capability.concurrencySafety(inv.input);
      switch (safety) {
        case 'safe':
          safe.push(inv);
          break;
        case 'path_scoped':
          pathScoped.push(inv);
          break;
        case 'exclusive':
          exclusive.push(inv);
          break;
      }
    }

    return { safe, pathScoped, exclusive };
  }

  /** 按 scopePath 分桶 */
  private groupByPath(
    invocations: readonly CapabilityInvocation[],
    _context: ExecutionContext,
  ): Map<string, CapabilityInvocation[]> {
    const buckets = new Map<string, CapabilityInvocation[]>();
    for (const inv of invocations) {
      // 从 input 中提取路径标识（约定：objectId 或 path 字段）
      const path = (inv.input['objectId'] as string)
        ?? (inv.input['path'] as string)
        ?? '__default__';
      const bucket = buckets.get(path) ?? [];
      bucket.push(inv);
      buckets.set(path, bucket);
    }
    return buckets;
  }
}
```

### 4.2 混合并发调度图解

```
时间轴 →

批量调用: [search-wiki, search-arxiv, read-object-A, update-object-A, delete-workspace]
         ├── safe ────────────┤  ├── path_scoped ──┤  ├── exclusive ──┤

Phase 1 (并行):
  ┌─ search-wiki ──────────────────┐  (safe)
  ├─ search-arxiv ─────────────────┤  (safe)
  └─ read-object-A → update-obj-A ┘  (path_scoped, 同路径串行)

Phase 2 (串行, 等 Phase 1 全部完成):
  ┌─ delete-workspace ─────────────┐  (exclusive)
```

### 4.3 执行预算追踪

BudgetTracker 追踪三个维度的预算消耗：

| 维度 | 初始值来源 | 消耗计算 | 超限行为 |
|------|-----------|---------|---------|
| **Token** | `OrchestratorConfig` 或 session 级配置 | 每次能力执行后，按结果字符数 / 4 估算 | 跳过后续能力，yield `budget_warning` |
| **时间** | session 级配置（默认 5 分钟） | 实际 `Date.now()` 差值 | 同上 |
| **金额** | 用户设置的 session 花费上限 | 按 `costProfile.estimatedCostCents` 预扣 | yield `budget_warning`，需用户确认后续 |

**预算退还**：当能力调用被 Safety Gate 拒绝或审批否决时，`BudgetTracker.refund()` 退还预扣的预算。

### 4.4 结果预算控制

借鉴 Hermes 的分层结果预算系统：

1. **无限制能力**（白名单）：如 `reading.read-object`，结果不截断（用户显式请求的内容应完整返回）
2. **标准限制**：结果超过 50K 字符时，压缩为 5K 摘要 + 引用
3. **聚合限制**：单轮所有能力结果累计超过 200K 字符时，对最长结果做摘要

压缩后的结果带引用标记，让 LLM 知道完整结果可用。后续 Agent 可通过 `reading.read-capability-output` 能力按需回溯完整结果。

## 5. 迁移策略

### 5.1 渐进迁移原则

迁移必须是**渐进式**的，不能一次性替换所有消费者。核心原则：

1. **新旧共存**：`ToolRegistry` 标记 `@deprecated` 但继续工作
2. **适配层桥接**：旧 `ToolDefinition` 可通过适配层注册为 `Capability`
3. **消费者逐步切换**：Orchestrator、DomainAgentConfig 等消费者按优先级逐步迁移
4. **测试双跑**：迁移期间旧测试和新测试同时运行，确保行为一致

### 5.2 ToolDefinition → Capability 适配层

```typescript
// packages/agent-core/src/capability-compat.ts

import { z } from 'zod';
import type { ToolDefinition, ToolHandler } from './types.js';
import type { Capability, CapabilityEvent } from './capability-types.js';
import { defineCapability } from './capability-defaults.js';

/**
 * 将旧的 ToolDefinition + ToolHandler 适配为新的 Capability 接口。
 *
 * 适配规则：
 * - 静态 riskLevel → 返回常量的函数
 * - 静态 approvalPolicy → 返回常量的函数
 * - concurrencySafety → 由 riskLevel 推导（R0-read → safe, 其余 → exclusive）
 * - inputSchema → z.record(z.unknown())（松散 schema）
 * - outputSchema → z.string()（旧 handler 返回 string）
 * - execute → 包装旧 handler 为 AsyncGenerator
 */
export function adaptToolToCapability(
  definition: ToolDefinition,
  handler: ToolHandler,
): Capability<Record<string, unknown>, string> {
  const derivedConcurrency = definition.riskLevel === 'R0-read' ? 'safe' as const : 'exclusive' as const;
  const capabilityId = `legacy.${definition.domain}.${definition.name}`;

  return defineCapability<Record<string, unknown>, string>({
    id: capabilityId,
    domain: definition.domain,
    description: definition.description,
    inputSchema: z.record(z.unknown()),
    outputSchema: z.string(),
    riskLevel: () => definition.riskLevel,
    approvalPolicy: () => definition.approvalPolicy,
    concurrencySafety: () => derivedConcurrency,
    executionMode: definition.executionMode,
    scopeLimit: definition.scopeLimit,
    costProfile: {
      estimatedTokens: { min: 0, max: 5000 },
      estimatedDurationMs: { min: 0, max: 10000 },
      estimatedCostCents: 0,
      externalBilling: false,
    },
    dataBoundary: definition.dataBoundary,
    async *execute(input, _context) {
      yield {
        kind: 'started',
        capabilityId,
        timestamp: new Date().toISOString(),
      };
      const result = await handler(input);
      yield {
        kind: 'completed',
        capabilityId,
        timestamp: new Date().toISOString(),
        progress: 1.0,
      };
      return result;
    },
  });
}

/**
 * 将整个 ToolRegistry 的内容批量迁移到 CapabilityRegistry。
 */
export function migrateToolRegistry(
  toolRegistry: import('./tool-registry.js').ToolRegistry,
  capabilityRegistry: import('./capability-registry.js').CapabilityRegistry,
): { migrated: number; failed: string[] } {
  const failed: string[] = [];
  let migrated = 0;

  for (const name of toolRegistry.getAllNames()) {
    const entry = toolRegistry.get(name);
    if (!entry) continue;
    try {
      const capability = adaptToolToCapability(entry.definition, entry.handler);
      capabilityRegistry.register(capability, { source: 'internal' });
      migrated++;
    } catch (err) {
      failed.push(name);
    }
  }

  return { migrated, failed };
}
```

## 附录 A：与 M1（事件流管道）的接口约定

M2 依赖 M1 定义的事件类型体系。`CapabilityEvent` 是 M1 事件层级中的叶子节点：

```
OrchestratorEvent
  └── AgentEvent
        └── CapabilityEvent  ← M2 定义
```

M2 的 `StreamingCapabilityExecutor.executeBatch()` 返回 `AsyncGenerator<CapabilityEvent>`，由 M1 的 Agent 执行层消费并向上传播。

如果 M1 尚未完成，M2 可以先独立开发：
- `CapabilityEvent` 在 M2 内部自行定义
- M1 完成后，统一到 M1 的事件类型体系中

## 附录 B：与 @orbit/capability-core 的关系

当前 `@orbit/capability-core` 包定义了 `CapabilityManifest`、`CapabilityPermission` 等类型（36 行）。这是一个**应用级能力声明**，关注的是"应用可以做什么"（如 search、sync、import）。

M2 的 `Capability<Input, Output>` 是 **Agent 级能力声明**，关注的是"Agent 可以调用什么工具"。两者是不同层次的抽象：

```
@orbit/capability-core    → 应用能力宣言（"Orbit 支持搜索"）
@orbit/agent-core/Capability → Agent 执行契约（"Agent 可以调用 research.search.external"）
```

两者的关联：一个 `capability-core.CapabilityManifest` 可以对应多个 `agent-core.Capability`。例如，`search` CapabilityManifest 可以对应 `research.search.local`、`research.search.external`、`research.search.workspace` 三个 Agent Capability。

迁移路径：保持 `@orbit/capability-core` 不变（它服务于应用层），M2 的类型定义在 `@orbit/agent-core` 内部。未来如有需要，可以在 `capability-core` 中增加从 `CapabilityManifest` → `Capability[]` 的映射函数。
