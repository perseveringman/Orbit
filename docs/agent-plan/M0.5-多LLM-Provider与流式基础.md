## 0. 问题总览与设计原则

### 当前痛点

| # | 问题 | 影响 | 现状代码 |
|---|------|------|----------|
| P1 | 仅抽象 `LLMAdapter` 接口 | 无具体实现，使用方必须自行编写 Provider 对接层 | `llm-adapter.ts` 只定义 `chatCompletion()` |
| P2 | 硬编码 OpenAI 格式 | `toOpenAIMessages` / `fromOpenAIResponse` 耦合 OpenAI 协议 | `llm-adapter.ts` 导出 OpenAI 转换器 |
| P3 | 模型写死 `gpt-4o` | 无法按任务/成本/能力动态选择模型 | `OrchestratorConfig.defaultModel` |
| P4 | 无流式支持 | 首 Token 延迟高、无法实时渲染、无法实现 thinking-tokens | `chatCompletion()` 返回完整 `Promise` |
| P5 | 无凭证管理 | 单 key 无法轮换，无法应对 rate-limit | 无相关代码 |
| P6 | 无成本追踪 | 用户无法感知 token 消耗与费用 | `TokenUsage` 仅透传不计价 |
| P7 | 无速率限制感知 | 429 错误时无自动恢复策略 | 无相关代码 |

### 设计原则

1. **Provider-Agnostic**：核心逻辑不依赖任何特定 Provider 协议；OpenAI / Anthropic / Ollama 格式转换仅存于各自 Provider 内部
2. **Stream-First**：所有 Provider 必须同时实现 `chatCompletion` 和 `chatCompletionStream`；非流式调用为流式的 `collect()` 语法糖
3. **Zero-Config 可用**：从环境变量自动检测可用 Provider，无需手动配置即可启动
4. **Fail-Closed 安全**：凭证不可逆序列化、不进入日志、不暴露给 Renderer 进程
5. **渐进增强**：成本追踪、速率限制等为可选层（decorator 模式），不影响核心 chat 路径
6. **Local-First 兼容**：OllamaProvider 支持完全离线场景，与 Orbit 的 local-first 哲学一致

### Hermes-Agent 参考对比

| 能力 | hermes-agent | Orbit 当前 | M0.5 目标 |
|------|-------------|-----------|----------|
| Provider 数量 | 7+（OpenAI, Anthropic, OpenRouter, Nous, z.ai, Kimi, MiniMax） | 0（仅抽象接口） | 3 内置 + 自定义端点 |
| 凭证轮换 | fill-first / round-robin / random / least-used | 无 | priority / round-robin / least-used |
| 辅助 Provider 链 | 支持（vision, compression 侧任务） | 无 | M1 阶段支持 |
| 模型元数据库 | context window / cost / capability flags | 无 | 完整支持 |
| 速率限制追踪 | 响应头解析 | 无 | 完整支持 |
| 成本估算 | 逐请求 | 无 | 逐请求 + 会话/全局预算 |
| Provider 自动检测 | 环境变量 | 无 | 环境变量 + Electron keychain |

## 1. Provider 抽象层

### 1.1 核心接口

```typescript
// packages/agent-core/src/llm-provider.ts

import type {
  ChatCompletionRequest,
  ChatCompletionResponse,
} from './types.js';

// ─── Stream Chunk 类型 ───────────────────────────────────────

/** 流式响应的分片类型（discriminated union） */
export type StreamChunk =
  | { type: 'text-delta';      delta: string }
  | { type: 'thinking-delta';  delta: string }
  | { type: 'tool-call-start'; index: number; id: string; name: string }
  | { type: 'tool-call-delta'; index: number; arguments: string }
  | { type: 'tool-call-end';   index: number }
  | { type: 'usage';           usage: StreamUsage }
  | { type: 'done';            finishReason: FinishReason };

export interface StreamUsage {
  promptTokens: number;
  completionTokens: number;
  totalTokens: number;
  /** Anthropic cache-read / cache-write tokens */
  cacheReadTokens?: number;
  cacheWriteTokens?: number;
}

export type FinishReason = 'stop' | 'tool_calls' | 'length' | 'content_filter';

// ─── Provider 接口 ───────────────────────────────────────────

/** LLM Provider 必须实现的核心接口 */
export interface LLMProvider {
  /** Provider 唯一标识，如 'openai', 'anthropic', 'ollama' */
  readonly id: string;

  /** 人类可读的显示名 */
  readonly displayName: string;

  /** 该 Provider 支持的模型列表（可动态获取） */
  listModels(): Promise<string[]>;

  /** 非流式调用 —— 默认实现为 stream + collect */
  chatCompletion(
    request: ChatCompletionRequest,
  ): Promise<ChatCompletionResponse>;

  /** 流式调用 —— 所有 Provider 必须原生实现 */
  chatCompletionStream(
    request: ChatCompletionRequest,
  ): AsyncGenerator<StreamChunk, void, undefined>;

  /** 检查该 Provider 是否已就绪（凭证可用且端点可达） */
  isAvailable(): Promise<boolean>;
}
```

### 1.4 内置 Provider 实现

#### OpenAIProvider

```typescript
// packages/agent-core/src/providers/openai-provider.ts

/**
 * OpenAI 兼容 Provider
 *
 * 同时支持 OpenAI 官方 API 和 OpenAI-compatible 端点
 * （OpenRouter、vLLM、LocalAI 等均可通过 baseUrl 配置接入）
 *
 * 格式转换逻辑：
 * - toOpenAIMessages()      从现有 llm-adapter.ts 迁入
 * - fromOpenAIResponse()    从现有 llm-adapter.ts 迁入
 * - toOpenAITools()         从现有 llm-adapter.ts 迁入
 * - parseSSEStream()        新增：解析 text/event-stream
 */
export class OpenAIProvider implements LLMProvider {
  readonly id = 'openai';
  readonly displayName = 'OpenAI';

  constructor(
    private config: ProviderConfig,
    private credentialPool: CredentialPool,
  ) {}

  async chatCompletion(req: ChatCompletionRequest): Promise<ChatCompletionResponse> {
    // 1. credentialPool.acquire('openai') 获取 API key
    // 2. toOpenAIMessages(req.messages) 转换消息格式
    // 3. fetch POST /v1/chat/completions { stream: false }
    // 4. fromOpenAIResponse(json) 转回 Orbit 格式
    // 5. credentialPool.release() 归还凭证
  }

  async *chatCompletionStream(req: ChatCompletionRequest): AsyncGenerator<StreamChunk> {
    // 1. 同上获取凭证与转换
    // 2. fetch POST /v1/chat/completions { stream: true, stream_options: { include_usage: true } }
    // 3. parseSSEStream(response.body) → yield StreamChunk
    // 4. 解析 [DONE] 事件 → yield { type: 'done' }
  }
}
```

## 2. 凭证管理 (CredentialPool)

### 2.1 设计思路

凭证池管理多个 API Key / Token 的生命周期。每个 Provider 可持有多组凭证，支持轮换策略以分散 rate-limit 压力，并在凭证失效时自动降级。

### 2.2 核心接口

```typescript
// packages/agent-core/src/credential-pool.ts

/** 单条凭证 */
export interface Credential {
  /** 凭证唯一 ID（内部标识，非 API Key） */
  id: string;
  /** 所属 Provider ID */
  providerId: string;
  /** API Key 或 Bearer Token（运行时明文，存储时加密） */
  secret: string;
  /** 凭证来源 */
  source: 'env' | 'keychain' | 'config' | 'oauth';
  /** 是否已标记为失效 */
  disabled: boolean;
  /** 最近一次使用时间 */
  lastUsedAt?: number;
  /** 累计使用次数 */
  useCount: number;
  /** OAuth token 过期时间（仅 oauth 来源） */
  expiresAt?: number;
}

/** 轮换策略 */
export type RotationStrategy = 'priority' | 'round-robin' | 'least-used';

/** 凭证池配置 */
export interface CredentialPoolConfig {
  /** 默认轮换策略 */
  defaultStrategy: RotationStrategy;
  /** 按 Provider 覆盖策略 */
  providerStrategies?: Record<string, RotationStrategy>;
  /** 环境变量自动检测映射 */
  envMapping?: Record<string, string>;
}

/** 凭证池 */
export class CredentialPool {
  private credentials = new Map<string, Credential[]>();
  private strategy: RotationStrategy;

  constructor(config: CredentialPoolConfig);

  /** 添加凭证 */
  add(credential: Omit<Credential, 'id' | 'useCount' | 'disabled'>): string;

  /** 获取下一个可用凭证（按策略选择） */
  acquire(providerId: string): Credential | null;

  /** 归还凭证（更新使用统计） */
  release(credentialId: string): void;

  /** 标记凭证失效（如 401 错误后） */
  disable(credentialId: string, reason: string): void;

  /** 从环境变量自动检测并添加凭证 */
  detectFromEnv(): DetectionResult;

  /** 从 Electron keychain 加载凭证（Renderer → Main IPC） */
  loadFromKeychain?(): Promise<void>;

  /** 刷新 OAuth token（如快过期时自动刷新） */
  refreshOAuthToken?(credentialId: string): Promise<void>;

  /** 获取指定 Provider 的可用凭证数量 */
  availableCount(providerId: string): number;
}

export interface DetectionResult {
  detected: Array<{ providerId: string; source: string }>;
  missing: string[];
}
```

### 2.3 环境变量自动检测

默认检测映射表：

| Provider | 环境变量 | 备注 |
|----------|---------|------|
| `openai` | `OPENAI_API_KEY` | 也兼容 `OPENAI_KEY` |
| `anthropic` | `ANTHROPIC_API_KEY` | — |
| `ollama` | —（无需凭证） | 检测端点可达性 |
| 自定义 | `LLM_API_KEY` | 通用 fallback |

```typescript
// 默认 envMapping
const DEFAULT_ENV_MAPPING: Record<string, string> = {
  openai: 'OPENAI_API_KEY',
  anthropic: 'ANTHROPIC_API_KEY',
};
```

### 2.4 轮换策略详解

| 策略 | 行为 | 适用场景 |
|------|------|---------|
| `priority` | 始终使用第一个可用凭证，故障后切换到下一个 | 单主 key + 备用 key |
| `round-robin` | 循环轮换，均匀分配请求 | 多 key 分散 rate-limit |
| `least-used` | 选择 `useCount` 最低的凭证 | 均衡长期使用量 |

### 2.5 安全考量

- **内存保护**：`Credential.secret` 仅在 `acquire()` 返回值中暴露，不进入序列化路径
- **日志脱敏**：所有日志输出中 API Key 自动替换为 `sk-...xxxx`（仅显示后 4 位）
- **Electron 安全**：Keychain 凭证通过 `contextBridge` 暴露安全 API，Renderer 进程无法直接读取
- **不可逆导出**：`CredentialPool` 不提供 `listSecrets()` 方法，防止批量泄露

## 4. 流式响应 (Streaming)

### 4.1 设计思路

流式是 M0.5 的核心交付。所有 Provider 必须原生实现 `chatCompletionStream()`，返回 `AsyncGenerator<StreamChunk>`。非流式 `chatCompletion()` 为流式的便利封装。

### 4.2 StreamChunk 类型体系

```typescript
/**
 * StreamChunk 是流式响应的最小原子单位。
 *
 * 设计参考：
 * - OpenAI: server-sent events with choices[0].delta
 * - Anthropic: content_block_start / content_block_delta / message_delta
 * - 统一为 Orbit 中间表示，各 Provider 负责转换
```

### 4.3 流式工具函数

```typescript
// packages/agent-core/src/stream-utils.ts

/**
 * 将 StreamChunk 流收集为完整的 ChatCompletionResponse
 *
 * 用于：
 * 1. chatCompletion() 非流式 API 的内部实现
 * 2. 测试中快速获取完整响应
 */
export async function collectStream(
  stream: AsyncGenerator<StreamChunk>,
): Promise<ChatCompletionResponse> {
  let content = '';
  const toolCalls: AgentToolCall[] = [];
  let usage: TokenUsage | undefined;
  let finishReason: FinishReason = 'stop';

  for await (const chunk of stream) {
    switch (chunk.type) {
      case 'text-delta':
        content += chunk.delta;
        break;
      case 'tool-call-start':
        toolCalls[chunk.index] = { id: chunk.id, name: chunk.name, arguments: '' };
        break;
      case 'tool-call-delta':
        toolCalls[chunk.index].arguments += chunk.arguments;
        break;
      case 'usage':
        usage = {
          promptTokens: chunk.usage.promptTokens,
          completionTokens: chunk.usage.completionTokens,
          totalTokens: chunk.usage.totalTokens,
        };
        break;
      case 'done':
        finishReason = chunk.finishReason;
        break;
    }
  }

  // 解析 tool call arguments JSON
  for (const tc of toolCalls) {
    if (tc) tc.arguments = JSON.parse(tc.arguments);
  }

  return {
    id: `chatcmpl-${Date.now()}`,
    choices: [{
      message: {
        role: 'assistant',
        content: content || null,
        toolCalls: toolCalls.length > 0 ? toolCalls.filter(Boolean) : undefined,
      },
      finishReason,
    }],
    usage: usage ?? { promptTokens: 0, completionTokens: 0, totalTokens: 0 },
  };
}

/**
 * SSE 解析器 —— 从 ReadableStream<Uint8Array> 解析 Server-Sent Events
 *
 * 兼容 OpenAI 和其他 OpenAI-compatible API 的 SSE 格式
 */
export async function* parseSSEStream(
  body: ReadableStream<Uint8Array>,
): AsyncGenerator<Record<string, unknown>> {
  const decoder = new TextDecoder();
  const reader = body.getReader();
  let buffer = '';

  try {
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;

      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split('\n');
      buffer = lines.pop() ?? '';

      for (const line of lines) {
        if (line.startsWith('data: ')) {
          const data = line.slice(6).trim();
          if (data === '[DONE]') return;
          yield JSON.parse(data);
        }
      }
    }
  } finally {
    reader.releaseLock();
  }
}

 */

  source: AsyncGenerator<T>,
```

### 5.2 成本计算公式

```typescript
function calculateCost(model: ModelMetadata, usage: StreamUsage): number {
  const { cost } = model;
  let total = 0;

  total += (usage.promptTokens / 1_000_000) * cost.inputPerMillion;
  total += (usage.completionTokens / 1_000_000) * cost.outputPerMillion;

  if (usage.cacheReadTokens && cost.cacheReadPerMillion) {
    total += (usage.cacheReadTokens / 1_000_000) * cost.cacheReadPerMillion;
  }
  if (usage.cacheWriteTokens && cost.cacheWritePerMillion) {
    total += (usage.cacheWriteTokens / 1_000_000) * cost.cacheWritePerMillion;
  }

  return total;
}
```


### 6.2 响应头解析

```typescript
/** 解析标准的速率限制响应头 */
function parseRateLimitHeaders(headers: Headers): RateLimitHeaders {
  return {
    limitRequests: parseIntHeader(headers, 'x-ratelimit-limit-requests'),
    remainingRequests: parseIntHeader(headers, 'x-ratelimit-remaining-requests'),
    limitTokens: parseIntHeader(headers, 'x-ratelimit-limit-tokens'),
    remainingTokens: parseIntHeader(headers, 'x-ratelimit-remaining-tokens'),
    resetRequests: headers.get('x-ratelimit-reset-requests') ?? undefined,
    resetTokens: headers.get('x-ratelimit-reset-tokens') ?? undefined,
    retryAfter: parseIntHeader(headers, 'retry-after'),
  };
}
```

### 6.3 指数退避策略

```typescript
/**
 * 指数退避 + 抖动
 *
 * baseMs = 1000, maxMs = 60000
 * attempt 0: 1s + jitter
 * attempt 1: 2s + jitter
 * attempt 2: 4s + jitter
 * attempt 3: 8s + jitter
 * ...
 * cap at 60s
 */
function calculateBackoff(attempt: number, baseMs = 1000, maxMs = 60_000): number {
  const exponential = Math.min(baseMs * Math.pow(2, attempt), maxMs);
  const jitter = Math.random() * exponential * 0.1; // 10% 抖动
  return exponential + jitter;
}
```

### 6.4 预判性凭证轮换

当 `remainingRequests` 低于阈值（默认 10%）时，RateLimiter 通知 CredentialPool 切换到下一个凭证，避免触及硬限制后的请求失败：

```
响应返回 x-ratelimit-remaining-requests: 5
  │
  ▼
RateLimiter.updateFromHeaders()
  │
  ├─ remaining / limit < 0.1 → shouldRotateCredential() returns true
  │                              → CredentialPool 切换到下一个凭证
  │
  └─ remaining / limit >= 0.1 → 继续使用当前凭证
```

### 7.2 错误恢复策略矩阵

| 错误类型 | 可重试 | 恢复策略 | 最大重试 |
|---------|-------|---------|---------|
| `auth` | 否 → 是（轮换后） | 标记凭证失效 → 轮换到下一个凭证 → 重试 | 凭证数 |
| `rate-limit` | 是 | 指数退避 → 凭证轮换 → Provider fallover | 3 |
| `context-overflow` | 否 → 是（压缩后） | 触发 ContextCompressor → 缩减上下文 → 重试 | 2 |
| `server-error` | 是 | 指数退避 → Provider fallover | 3 |
| `network` | 是 | 指数退避 → 检查网络 → Provider fallover | 3 |
| `content-filter` | 否 | 返回用户可读错误，建议修改输入 | 0 |
| `unknown` | 否 | 记录详细错误日志，返回通用错误 | 0 |

### 7.3 Provider Failover

当某个 Provider 连续失败超过阈值时，自动切换到备选 Provider：

```typescript
 * Failover 流程：

 * 1. OpenAI 连续 3 次 server-error
```

```typescript
    private llm: LLMAdapter,
    // ...

  const response = await this.llm.chatCompletion(request);

// 改造后
export class Orchestrator {
  constructor(
    private providerRegistry: ProviderRegistry,
    private modelRegistry: ModelRegistry,
    private costTracker: CostTracker,
    private rateLimiter: RateLimiter,
    // ...
  ) {}

  private async callLLM(
    request: ChatCompletionRequest,
    domainConfig: DomainAgentConfig,
  ): AsyncGenerator<StreamChunk> {
    // 1. 从 domainConfig.modelPreference 解析目标模型
    const model = this.modelRegistry.recommend(domainConfig.modelPreference);

    // 2. 检查预算
    const budget = this.costTracker.canProceed(sessionId);
    if (!budget.allowed && budget.alert?.level === 'critical') {
      throw new BudgetExceededError(budget.alert);
    }

    // 3. 检查速率限制
    const rateCheck = this.rateLimiter.canRequest(model.providerId, model.modelId);
    if (!rateCheck.allowed) {
      await sleep(rateCheck.waitMs);
    }

    // 4. 获取 Provider 并执行流式调用
    const provider = this.providerRegistry.resolve(model.providerId);
    yield* provider.chatCompletionStream({ ...request, model: model.modelId });
  }
}
```
